# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

资源耗尽攻击是指攻击者通过触发异常或发送大量请求，导致系统资源（内存、磁盘、连接等）被耗尽，从而造成服务拒绝或系统不稳定。

**本质问题**：
- 异常处理后资源未释放
- 缺少资源配额限制
- 缺少速率限制

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-400 | 资源消耗失控 |
| CWE-770 | 无速率限制的资源分配 |
| CWE-636 | 未安全失败 |

## 2.2 攻击常见于哪些业务场景

| 场景 | 风险点 | 潜在危害 |
|-----|-------|---------|
| 文件上传 | 文件句柄未释放 | 文件描述符耗尽 |
| 数据库操作 | 连接未关闭 | 连接池耗尽 |
| 图像处理 | 内存未释放 | 内存溢出 |
| 会话管理 | 会话未清理 | 会话存储耗尽 |
| 日志记录 | 日志填满磁盘 | 磁盘空间耗尽 |

## 2.3 漏洞发现方法

### 2.3.1 文件句柄泄露检测

```bash

    curl -X POST https://target.com/upload \
        -F "file=@malformed_file" &

```

### 2.3.4 自动化测试脚本

```python
import requests
import time

def test_resource_leak():
    for i in range(1000):
        try:
            # 发送会触发异常的请求
            response = requests.post(
                'https://target.com/api/process',
                json={'data': 'x' * 1000000}  # 大数据
            )
        except:
            pass
        
        if i % 100 == 0:
            print(f"Sent {i} requests")
            time.sleep(1)
```

## 2.4 漏洞利用方法

### 2.4.1 文件描述符耗尽

```bash
# 持续上传文件触发异常
# 每次异常泄露一个文件句柄

# 当文件句柄耗尽时：
# - 无法创建新文件
# - 无法接受新连接
# - 服务崩溃
```

### 2.4.2 数据库连接池耗尽

```bash
# 发送大量并发查询
# 每个查询触发异常并泄露连接

# 当连接池耗尽时：
# - 新查询无法执行
# - 应用报错
# - 服务不可用
```

### 2.4.3 内存耗尽

```bash
# 发送大对象处理请求
# 触发异常但内存未释放

# 当内存耗尽时：
# - OOM Killer 杀死进程
# - 服务崩溃
# - 需要重启
```

### 2.4.4 磁盘空间耗尽

```bash
# 触发大量错误日志记录
# 或上传大量文件

# 当磁盘满时：
# - 无法写入新数据
# - 服务异常
# - 可能影响其他服务
```

## 3.3 修复建议

1. **正确异常处理** - 使用 try-finally 确保资源释放
2. **资源配额** - 设置资源使用上限
3. **速率限制** - 限制请求频率
4. **监控告警** - 监控资源使用并告警
5. **自动恢复** - 实现资源自动回收
