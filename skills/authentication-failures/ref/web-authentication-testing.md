# **第二部分：核心渗透技术专题**

## 专题一：Web 认证测试

### 2.1 技术介绍

Web 认证测试是指对 Web 应用的身份验证机制进行全面的安全评估。其本质是**验证 Web 应用能否正确识别和验证用户身份，防止未授权访问**。

Web 认证核心组件：

| **组件** | **作用** | **常见实现** |
| :--- | :--- | :--- |
| 认证凭证 | 用户身份证明 | 用户名/密码、生物特征 |
| Session 管理 | 维持认证状态 | Session ID、Cookie |
| 令牌机制 | 无状态认证 | JWT、OAuth Token |
| 多因素认证 | 增强认证强度 | OTP、硬件令牌 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **用户登录** | 登录表单、SSO 登录 | 暴力破解、凭证泄露 |
| **会话管理** | Cookie、Session 处理 | 会话劫持、固定 |
| **密码管理** | 修改密码、密码重置 | 逻辑缺陷、信息泄露 |
| **注册流程** | 新用户注册 | 账户枚举、自动化注册 |
| **退出登录** | 注销功能 | 会话未正确失效 |
| **记住我** | 自动登录功能 | 令牌可预测、长期有效 |
| **多设备登录** | 并发会话管理 | 会话冲突、权限混乱 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **登录功能测试**
  - 测试默认凭证（admin/admin、root/root 等）
  - 测试 SQL 注入（`' OR '1'='1`）
  - 测试暴力破解（无速率限制）
  - 测试账户枚举（不同错误消息）
  - 测试 HTTP 方法绕过（GET vs POST）

- **会话管理测试**
  - 检查 Session ID 的随机性和长度
  - 测试会话超时机制
  - 测试并发会话限制
  - 测试注销后会话失效
  - 测试 Cookie 安全属性（HttpOnly、Secure、SameSite）

- **认证绕过测试**
  - 直接访问受保护页面
  - 修改 URL 参数绕过认证
  - 测试 HTTP 头注入（X-Auth-User 等）
  - 测试路径遍历绕过

#### 2.3.2 白盒测试

- **代码审计**
  - 检查密码存储方式（是否使用 bcrypt、Argon2 等）
  - 审计 Session ID 生成逻辑
  - 检查认证逻辑的完整性
  - 查找硬编码凭证

- **配置检查**
  - 检查 HTTPS 强制实施
  - 检查 Cookie 配置
  - 检查会话超时设置
  - 检查认证日志记录

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

| **操作** | **方法** | **目的** |
| :--- | :--- | :--- |
| 技术栈识别 | Wappalyzer、BuiltWith | 了解框架和中间件 |
| 登录入口发现 | 爬虫、目录扫描 | 发现所有认证入口 |
| 会话机制分析 | 浏览器开发者工具 | 分析 Cookie 和 Session |
| 认证流程映射 | Burp Suite 记录 | 绘制认证流程图 |

#### 2.4.2 凭证攻击

```bash
# 使用 Hydra 进行暴力破解
hydra -L usernames.txt -P passwords.txt \
  https://target.com/login http-post-form \
  "/login:username=^USER^&password=^PASS^:Invalid"

# 使用 Burp Intruder
# 设置 Payload 位置，分析响应差异
```

#### 2.4.3 会话攻击

- **会话劫持** - 窃取有效 Session ID 冒充用户
- **会话固定** - 诱导用户使用攻击者设定的 Session ID
- **会话重放** - 重放捕获的认证请求

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过前端验证

- **禁用 JavaScript** - 绕过客户端验证
- **直接发送请求** - 绕过前端表单逻辑
- **修改隐藏字段** - 篡改表单中的隐藏参数

#### 2.5.2 绕过 WAF/防护

- **请求编码** - URL 编码、Base64 编码
- **分块传输** - 使用 Transfer-Encoding: chunked
- **HTTP 方法变异** - 尝试不同 HTTP 方法

#### 2.5.3 绕过速率限制

- **IP 轮换** - 使用代理池
- **用户名变异** - 尝试不同用户名组合
- **时间分散** - 延长请求间隔

# **第三部分：附录**

## 3.1 Web 认证测试检查清单

| **检查项** | **测试方法** | **预期结果** |
| :--- | :--- | :--- |
| 密码复杂度策略 | 尝试设置弱密码 | 应拒绝弱密码 |
| 账户锁定机制 | 多次失败登录 | 应临时锁定账户 |
| 会话超时 | 等待超时时间 | 应自动注销 |
| 注销功能 | 点击注销后访问页面 | 应拒绝访问 |
| Cookie 安全属性 | 检查 Cookie 属性 | 应设置 HttpOnly、Secure |
| HTTPS 强制 | 尝试 HTTP 访问 | 应重定向到 HTTPS |
| 错误消息 | 输入错误凭证 | 不应泄露具体错误原因 |
| 验证码机制 | 多次失败后 | 应要求验证码 |

## 3.2 Web 认证测试工具

| **工具名称** | **用途** | **命令示例** |
| :--- | :--- | :--- |
| **Burp Suite** | 拦截/修改请求 | 手动测试、Intruder |
| **OWASP ZAP** | 自动化扫描 | Active Scan |
| **Hydra** | 暴力破解 | `hydra -l user -P pass.txt target` |
| **SQLmap** | SQL 注入测试 | `sqlmap -u "url" --forms` |
| **Nikto** | Web 扫描 | `nikto -h target` |
| **Wfuzz** | Web Fuzzing | `wfuzz -c -z file,users.txt` |

## 3.3 修复建议

1. **强密码策略** - 要求最小长度、复杂度
2. **多因素认证** - 对敏感操作启用 MFA
3. **安全会话管理** - 使用安全的 Session ID 生成
4. **HTTPS 强制** - 所有认证流量使用 HTTPS
5. **速率限制** - 实施登录尝试限制
6. **账户锁定** - 多次失败后临时锁定
7. **安全 Cookie** - 设置 HttpOnly、Secure、SameSite
8. **审计日志** - 记录所有认证事件
9. **输入验证** - 对所有输入进行验证和转义
10. **定期更新** - 保持认证库和框架更新
