# **第二部分：核心渗透技术专题**

## 专题：MFA 绕过攻击

## 2.1 技术介绍
- **漏洞原理：** MFA 绕过是指攻击者通过利用 MFA 实现中的逻辑缺陷、配置错误或社会工程手段，在未完成第二因素认证的情况下获得账户访问权限。
- **本质：** 多因素认证流程存在设计缺陷或实现问题，导致额外的认证因素可被跳过、预测、重放或绕过。

| **绕过类型** | **描述** | **常见原因** |
| :--- | :--- | :--- |
| **MFA 疲劳攻击** | 持续发送 MFA 请求直到用户误批准 | 用户安全意识不足，无请求频率限制 |
| **MFA 逻辑绕过** | 直接访问 MFA 后的页面或功能 | 服务端未严格验证 MFA 完成状态 |
| **验证码预测** | 预测或暴力破解验证码 | 验证码熵值低、无速率限制 |
| **验证码重放** | 重用已使用过的验证码 | 未实施一次性验证机制 |
| **SIM 交换攻击** | 接管受害者手机号码 | 运营商身份验证流程薄弱 |
| **推送通知轰炸** | 大量推送直到用户批准或拒绝错误 | 无推送频率限制和异常检测 |

## 2.2 攻击常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **登录流程** | 用户名密码后要求 MFA | MFA 验证状态未在服务端严格检查 |
| **敏感操作** | 转账、改密、绑定设备前 MFA | 可绕过 MFA 直接执行敏感操作 |
| **设备信任** | 信任此设备/30 天内不再询问 | 信任机制实现不当可被利用 |
| **备用验证** | 短信/TOTP/邮件多种 MFA 方式 | 可在不同方式间切换选择最弱的 |
| **密码重置** | 通过 MFA 重置密码 | 重置流程中 MFA 验证可被绕过 |
| **账户恢复** | 通过备用方式恢复账户 | 恢复流程验证强度不足 |

## 2.3 漏洞发现方法

### 2.3.1 黑盒测试

- **MFA 状态检测**
  - 完成 MFA 后记录所有 Cookie 和 Token
  - 清除 MFA 相关 Cookie 后访问受保护页面
  - 使用旧会话 Token 访问 MFA 后才能访问的接口

- **直接绕过测试**
  - 输入正确密码后，直接访问 MFA 后的目标 URL
  - 修改响应包中的 MFA 验证状态：`mfa_verified=false` 改为 `true`
  - 删除或修改 MFA 相关的请求参数

- **验证码测试**
  - 测试验证码长度和复杂度（4 位数字最弱）
  - 测试验证码是否可重复使用
  - 测试验证码有效期（尝试延迟使用）
  - 测试速率限制（连续尝试多个验证码）

- **多 MFA 方式测试**
  - 识别所有可用的 MFA 方式
  - 测试能否切换到更弱的验证方式
  - 测试能否禁用或跳过某些 MFA 方式

### 2.3.2 白盒测试

- **代码审计要点**
  - 检查 MFA 验证状态的存储和检查逻辑
  - 查看验证码生成算法（随机性、熵值）
  - 检查验证码一次性使用机制
  - 审查 MFA 绕过保护逻辑（速率限制、异常检测）

- **关键代码模式**
  ```python
  # 危险模式：仅检查 MFA 参数存在性
  if request.params.get('mfa_verified'):
      grant_access()
  
  # 危险模式：验证码可重复使用
  if stored_code == user_input:  # 未标记为已使用
      verify_success()
  ```

## 2.4 漏洞利用方法

### 2.4.1 基础信息收集

- **MFA 机制识别**
  - 确定使用的 MFA 类型（短信、TOTP、推送、硬件令牌）
  - 识别 MFA 触发条件和验证流程
  - 收集 MFA 相关的 Cookie、Token、参数

- **验证码特征分析**
  - 分析验证码长度、字符集、格式
  - 测试验证码与时间/会话的关联性
  - 收集多个验证码样本分析模式

### 2.4.2 MFA 疲劳攻击

- **攻击步骤**
  1. 获取目标用户名和密码（通过其他手段）
  2. 发起登录请求触发 MFA
  3. 持续重复登录请求，产生大量 MFA 通知
  4. 等待用户因疲劳而误批准，或拒绝时误操作

- **自动化脚本示例**
  ```bash
  # 循环发起登录请求
  for i in {1..100}; do
      curl -X POST https://target.com/login \
           -d "username=victim&password=known_pass"
      sleep 2
  done
  ```

### 2.4.3 验证码暴力破解

- **4 位数字验证码**（10000 种可能）
  ```python
  for code in range(10000):
      response = requests.post(url, data={'code': f'{code:04d}'})
      if 'success' in response.text:
          print(f'Found code: {code}')
          break
  ```

- **6 位数字验证码**（1000000 种可能）
  - 需要绕过速率限制（IP 轮换、参数变异）
  - 可结合用户名枚举缩小范围

### 2.4.4 信任机制滥用

- **设备信任绕过**
  - 修改 User-Agent 模拟已信任设备
  - 复制已信任设备的 Cookie/Token
  - 利用"信任此设备"功能的逻辑缺陷

- **记住我功能滥用**
  - 分析"记住我"Token 的生成逻辑
  - 测试 Token 是否与 MFA 状态绑定
  - 尝试重放或伪造长期有效的 Token

### 2.4.5 SIM 交换攻击

- **攻击流程**
  1. 收集目标个人信息（社交媒体、数据泄露）
  2. 联系运营商冒充受害者申请 SIM 卡补办
  3. 获得目标手机号的控制权
  4. 接收短信验证码重置密码或登录

## 2.5 漏洞利用绕过方法

### 2.5.1 绕过速率限制

- **IP 轮换技术**
  - 使用代理池（住宅代理、移动代理）
  - 利用云函数（AWS Lambda、Azure Functions）动态 IP
  - 使用 Tor 网络轮换出口节点

- **参数变异技术**
  - 在请求中添加随机参数：`_rand=随机值`
  - 使用不同的参数名：`code`、`otp`、`token`、`mfa_code`
  - 修改 User-Agent 和其他请求头

### 2.5.2 绕过设备指纹检测

- **浏览器指纹伪造**
  - 使用自动化工具复制完整浏览器环境
  - 修改 Canvas、WebGL、Audio 等指纹特征
  - 保持一致的时区、语言、字体设置

- **设备标识符篡改**
  - 修改或伪造设备 ID Cookie
  - 利用虚拟机/容器快速切换设备环境
  - 使用反检测浏览器（AdsPower、Multilogin）

### 2.5.3 社会工程辅助

- **钓鱼攻击获取 MFA**
  - 搭建钓鱼网站实时代理 MFA 请求
  - 使用 Evilginx 等工具进行中间人攻击
  - 发送伪造的 MFA 请求邮件/SMS

- **客服支持滥用**
  - 冒充用户联系客服要求禁用 MFA
  - 利用客服验证流程的薄弱环节
  - 通过社会工程获取账户恢复权限

# **第三部分：附录**

## 3.1 常用 Payload 速查表

| **类别** | **Payload 示例** | **说明** |
| :--- | :--- | :--- |
| **参数绕过** | `mfa_verified=true` | 伪造 MFA 验证状态 |
| **参数绕过** | `skip_mfa=true`、`bypass_mfa=1` | 尝试跳过 MFA |
| **Cookie 篡改** | `mfa_status=verified` | 修改 MFA 状态 Cookie |
| **Cookie 篡改** | `trusted_device=true` | 伪造信任设备标识 |
| **验证码爆破** | 0000-9999 (4 位) | 4 位数字验证码范围 |
| **验证码爆破** | 000000-999999 (6 位) | 6 位数字验证码范围 |
| **TOTP 工具** | `gap --secret=xxx` | 使用 gap 工具生成 TOTP |
| **TOTP 工具** | Python pyotp 库 | 编程生成 TOTP 验证码 |

## 3.2 MFA 测试检查清单

| **检查项** | **测试方法** | **安全要求** |
| :--- | :--- | :--- |
| **MFA 强制启用** | 尝试不启用 MFA 访问敏感功能 | 敏感操作应强制 MFA |
| **验证码强度** | 分析验证码长度和复杂度 | 至少 6 位，包含字母数字 |
| **速率限制** | 连续尝试多个验证码 | 应有失败次数限制 |
| **一次性使用** | 重用已使用的验证码 | 验证码应一次性有效 |
| **有效期检查** | 延迟使用验证码 | 应有合理有效期（30-60 秒） |
| **MFA 状态验证** | 绕过 MFA 步骤访问资源 | 服务端应严格验证 |
| **备用方式强度** | 比较不同 MFA 方式强度 | 所有方式都应足够安全 |
| **设备信任机制** | 测试信任设备功能 | 信任 Token 应安全绑定 |
