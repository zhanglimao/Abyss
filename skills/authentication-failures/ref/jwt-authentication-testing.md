# **第二部分：核心渗透技术专题**

## 专题一：JWT 认证测试

### 2.1 技术介绍

JWT 认证测试是指对 JSON Web Token 的生成、传输和验证进行全面的安全评估。其本质是**验证 JWT 实现能否安全地处理身份认证，防止令牌伪造、篡改和滥用**。

JWT 结构：

| **部分** | **描述** | **安全风险** |
| :--- | :--- | :--- |
| Header | 算法和令牌类型 | 算法篡改、密钥混淆 |
| Payload | 声明（用户信息、权限等） | 信息泄露、篡改 |
| Signature | 签名验证完整性 | 签名绕过、弱密钥 |

JWT 常见算法：
- **HS256/HS384/HS512** - HMAC with SHA（对称加密）
- **RS256/RS384/RS512** - RSA with SHA（非对称加密）
- **ES256/ES384/ES512** - ECDSA with SHA（椭圆曲线）
- **none** - 无签名（**危险**）

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **用户登录** | 登录后颁发 JWT | 令牌伪造、权限提升 |
| **API 认证** | Authorization: Bearer | 令牌验证不当 |
| **微服务通信** | 服务间 JWT 传递 | 信任链验证缺失 |
| **单点登录** | SSO JWT 断言 | 跨服务认证绕过 |
| **刷新令牌** | Refresh Token 机制 | 令牌无限刷新 |
| **第三方集成** | JWT 交换 | 令牌泄露、重放 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **令牌分析**
  - 解码 JWT 查看 header 和 payload
  - 检查算法类型
  - 检查有效期（exp、nbf、iat）
  - 检查敏感信息泄露

- **算法攻击测试**
  - 尝试将算法改为 `none`
  - 尝试 HS256/RS256 混淆攻击
  - 尝试弱密钥爆破
  - 尝试算法降级

- **令牌篡改测试**
  - 修改 payload 中的用户 ID
  - 修改权限/角色声明
  - 修改有效期
  - 重放过期令牌

- **令牌生命周期测试**
  - 测试令牌有效期
  - 测试刷新机制
  - 测试撤销机制
  - 测试并发会话

#### 2.3.2 白盒测试

- **代码审计**
  - 检查 JWT 生成逻辑
  - 审计签名验证代码
  - 检查密钥管理
  - 查找算法验证逻辑

- **配置检查**
  - 检查密钥强度和存储
  - 检查有效期配置
  - 检查允许的算法白名单
  - 检查日志记录

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

```bash
# 解码 JWT（使用 jwt.io 或命令行）
echo "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c" | cut -d'.' -f2 | base64 -d

# 使用 jwt-tool 分析
python3 jwt_tool.py <JWT_TOKEN>
```

#### 2.4.2 算法 None 攻击

```python
# 原始 JWT
# Header: {"alg":"RS256","typ":"JWT"}
# Payload: {"sub":"user","role":"user"}

# 攻击：修改为 none 算法
# Header: {"alg":"none","typ":"JWT"}
# Payload: {"sub":"admin","role":"admin"}
# Signature: (空)
```

#### 2.4.3 HS256/RS256 混淆攻击

```python
# 当服务端使用 RS256（非对称）但接受 HS256（对称）时
# 攻击者可以使用公钥作为 HMAC 密钥

# 1. 获取服务端公钥
# 2. 使用公钥作为 HMAC 密钥签名
# 3. 将算法改为 HS256
# 4. 服务端可能误用公钥验证 HMAC 签名
```

#### 2.4.4 弱密钥爆破

| **操作** | **方法** | **目的** |
| :--- | :--- | :--- |
| 字典攻击 | 使用常见密钥字典 | 破解弱密钥 |
| 规则爆破 | 使用密钥派生规则 | 破解模式密钥 |
| 暴力破解 | 短密钥穷举 | 破解短密钥 |

#### 2.4.5 JWT 攻击工具

```bash
# 使用 jwt-tool 进行攻击
python3 jwt_tool.py <JWT_TOKEN> -X a -p password.txt  # 密码爆破
python3 jwt_tool.py <JWT_TOKEN> -X n  # none 算法
python3 jwt_tool.py <JWT_TOKEN> -X k -pk public.pem  # 密钥混淆

# 使用 hashcat 破解
hashcat -m 16500 jwt.txt wordlist.txt
```

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过签名验证

- **算法篡改** - 改为 `none` 或弱算法
- **JWK 注入** - 在 header 中注入自定义 JWK
- **密钥混淆** - 利用非对称/对称算法混淆
- **KID 注入** - 修改 key ID 指向可控密钥

#### 2.5.2 绕过有效期检查

- **时间同步** - 修改系统时间
- **时区绕过** - 利用时区处理差异
- **边界测试** - 在有效期边界测试

#### 2.5.3 绕过撤销检查

- **离线使用** - 如果服务端未检查撤销
- **多服务重用** - 在不同服务间重用
- **快速使用** - 在撤销前快速使用

## 3.3 修复建议

1. **算法白名单** - 仅允许安全的非对称算法
2. **强制签名验证** - 始终验证签名
3. **强密钥** - 使用足够长度的随机密钥
4. **密钥轮换** - 定期轮换密钥
5. **短期有效** - 设置短的令牌有效期
6. **撤销机制** - 实现令牌撤销列表
7. **最小声明** - 仅包含必要声明
8. **HTTPS 传输** - 所有 JWT 使用 HTTPS 传输
9. **存储安全** - 客户端安全存储（HttpOnly Cookie）
10. **审计日志** - 记录所有 JWT 使用事件
