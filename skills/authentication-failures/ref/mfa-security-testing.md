# **第二部分：核心渗透技术专题**

## 专题一：MFA 安全测试

### 2.1 技术介绍

MFA（Multi-Factor Authentication）安全测试是指对多因素认证实现进行全面的安全评估。其本质是**验证 MFA 系统能否正确地要求并验证多个认证因素，防止绕过和暴力破解**。

MFA 常见类型：

| **类型** | **描述** | **安全风险** |
| :--- | :--- | :--- |
| 短信验证码 | 发送 OTP 到手机 | SIM 交换、拦截 |
| 邮件验证码 | 发送 OTP 到邮箱 | 邮箱接管 |
| TOTP | 时间同步一次性密码 | 种子泄露、重放 |
| HOTP | 计数器 OTP | 计数器同步问题 |
| 推送通知 | 移动应用推送确认 | 推送劫持、疲劳攻击 |
| 硬件令牌 | YubiKey 等 | 丢失、钓鱼 |
| 生物识别 | 指纹、面部 | 绕过、重放 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **登录 MFA** | 登录后要求第二因素 | 绕过、暴力破解 |
| **敏感操作** | 转账、修改密码前验证 | 验证逻辑缺陷 |
| **设备绑定** | 新设备登录要求 MFA | 设备指纹伪造 |
| **密码重置** | 重置前 MFA 验证 | 流程绕过 |
| **管理员登录** | 特权账户强制 MFA | 权限提升绕过 |
| **API 访问** | 敏感 API 要求 MFA | 验证缺失 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **MFA 绕过测试**
  - 尝试直接访问 MFA 后页面
  - 尝试修改 MFA 验证响应
  - 尝试删除/修改 MFA 相关参数
  - 尝试 HTTP 状态码修改

- **验证码暴力破解**
  - 测试验证码长度和复杂度
  - 测试速率限制
  - 测试账户锁定
  - 测试验证码有效期

- **重放攻击测试**
  - 尝试重复使用验证码
  - 尝试多会话重用
  - 尝试时间窗口外使用

- **逻辑缺陷测试**
  - 测试多用户验证码交叉使用
  - 测试验证码与用户绑定
  - 测试流程顺序绕过

#### 2.3.2 白盒测试

- **代码审计**
  - 检查 MFA 验证逻辑
  - 审计验证码生成算法
  - 检查速率限制实现
  - 查找硬编码绕过

- **配置检查**
  - 检查 TOTP 种子存储
  - 检查短信/邮件服务配置
  - 检查备份码生成
  - 检查恢复流程

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

```bash
# 拦截 MFA 请求
# 分析验证码长度、格式、有效期

# 测试速率限制
for i in {1..100}; do
  curl -X POST https://target.com/mfa/verify \
    -d "code=123456&session=$SESSION"
done

# 测试验证码有效期
# 等待不同时间后尝试使用
```

#### 2.4.2 MFA 绕过攻击

| **绕过方法** | **描述** | **成功率** |
| :--- | :--- | :--- |
| 直接访问 | 访问 MFA 后 URL | 中 |
| 响应篡改 | 修改验证响应为成功 | 中 - 高 |
| 参数删除 | 删除 MFA 验证参数 | 中 |
| 状态码修改 | 客户端修改响应状态码 | 低 |
| Cookie 注入 | 注入 MFA 成功 Cookie | 中 |
| JS 绕过 | 禁用前端验证 | 低 |

#### 2.4.3 暴力破解

```bash
# 使用 Hydra 暴力破解
hydra -l user -P codes.txt \
  https://target.com http-post-form \
  "/mfa/verify:code=^PASS^:Invalid"

# 使用 Burp Intruder
# 设置 Payload 为 000000-999999
# 分析响应差异
```

#### 2.4.4 TOTP 攻击

- **种子泄露** - 从数据库或备份获取种子
- **时间同步** - 利用时间窗口
- **钓鱼攻击** - 窃取实时验证码
- **SIM 交换** - 接管短信验证码

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过速率限制

- **IP 轮换** - 使用代理池
- **账户轮换** - 尝试多个账户
- **时间分散** - 延长尝试间隔
- **验证码变异** - 尝试不同格式

#### 2.5.2 绕过设备绑定

- **设备指纹伪造** - 修改设备标识
- **Cookie 复制** - 从已授权设备复制 Cookie
- **User-Agent 伪造** - 伪装成已知设备

#### 2.5.3 MFA 疲劳攻击

- **连续推送** - 发送大量推送通知
- **时间选择** - 在用户疲劳时攻击
- **社会工程** - 伪装成 IT 支持

# **第三部分：附录**

## 3.1 MFA 测试检查清单

| **检查项** | **测试方法** | **预期结果** |
| :--- | :--- | :--- |
| MFA 强制 | 跳过 MFA 步骤 | 应拒绝访问 |
| 验证码长度 | 分析验证码格式 | 应至少 6 位 |
| 速率限制 | 高频尝试 | 应限制并告警 |
| 账户锁定 | 多次失败 | 应临时锁定 |
| 验证码有效期 | 延迟使用 | 应拒绝过期码 |
| 重放保护 | 重复使用 | 应拒绝重用 |
| 用户绑定 | 交叉使用验证码 | 应拒绝 |
| 备份码 | 测试备份码机制 | 应安全生成 |

## 3.3 修复建议

1. **强制 MFA** - 对敏感操作强制要求 MFA
2. **强验证码** - 使用至少 6 位随机验证码
3. **速率限制** - 实施严格的尝试限制
4. **账户锁定** - 多次失败后锁定
5. **短期有效** - 验证码有效期不超过 5 分钟
6. **一次性使用** - 验证码使用后立即失效
7. **用户绑定** - 验证码与用户会话严格绑定
8. **备份码** - 提供安全的备份码机制
9. **审计日志** - 记录所有 MFA 事件
10. **防疲劳** - 限制推送频率，提供举报机制
