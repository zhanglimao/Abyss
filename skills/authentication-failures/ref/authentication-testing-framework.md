# **第二部分：核心渗透技术专题**

## 专题一：认证测试框架

### 2.1 技术介绍

认证测试框架是一个系统性的方法论，用于指导渗透测试人员全面评估身份认证系统的安全性。其本质是**通过结构化的测试流程，确保所有认证相关的安全风险都被充分评估**。

认证测试核心领域：

| **领域** | **描述** | **关键风险** |
| :--- | :--- | :--- |
| 凭证管理 | 密码、密钥的存储和处理 | 弱哈希、明文存储 |
| 会话管理 | Session、Token 的生命周期 | 劫持、固定、预测 |
| 多因素认证 | MFA/2FA 实现 | 绕过、暴力破解 |
| 联邦认证 | SSO、SAML、OAuth | 配置错误、信任滥用 |
| 账户恢复 | 密码重置、账户找回 | 逻辑缺陷、枚举 |
| 认证策略 | 密码策略、锁定策略 | 配置不当 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **测试重点** |
| :--- | :--- | :--- |
| **用户注册** | 新用户创建账户 | 密码策略、邮箱验证 |
| **用户登录** | 凭证验证 | 暴力破解、枚举、注入 |
| **会话管理** | Cookie、Token 处理 | 劫持、固定、超时 |
| **密码管理** | 修改、重置密码 | 逻辑缺陷、旧密码验证 |
| **多因素认证** | 2FA/MFA | 绕过、暴力破解 |
| **第三方登录** | OAuth、社交登录 | 重定向劫持、令牌泄露 |
| **API 认证** | API Key、JWT | 令牌验证、权限控制 |
| **账户恢复** | 忘记密码、安全问题 | 枚举、逻辑缺陷 |

### 2.3 漏洞探测方法

#### 2.3.1 测试规划阶段

```
认证测试范围定义
├── 资产识别
│   ├── 认证端点清单
│   ├── 用户类型清单
│   └── 认证方式清单
├── 威胁建模
│   ├── STRIDE 分析
│   ├── 攻击树分析
│   └── 风险优先级
└── 测试计划
    ├── 测试方法（黑盒/白盒）
    ├── 测试工具准备
    └── 测试时间安排
```

#### 2.3.2 信息收集阶段

- **应用映射**
  - 识别所有认证端点
  - 绘制认证流程图
  - 识别所有输入点
  - 识别所有会话处理点

- **技术栈识别**
  - 识别认证框架/库
  - 识别会话管理机制
  - 识别密码存储方式
  - 识别第三方认证集成

- **配置收集**
  - 收集公开的配置信息
  - 分析错误消息
  - 分析 HTTP 头信息
  - 分析 JavaScript 文件

#### 2.3.3 漏洞测试阶段

| **测试类别** | **测试项** | **参考文档** |
| :--- | :--- | :--- |
| 凭证测试 | 默认凭证、暴力破解 | `ref/default-credential-exploitation.md` |
| 会话测试 | 会话固定、劫持、预测 | `ref/session-management-testing.md` |
| MFA 测试 | 绕过、暴力破解 | `ref/mfa-security-testing.md` |
| JWT 测试 | 算法篡改、密钥爆破 | `ref/jwt-authentication-testing.md` |
| OAuth 测试 | 重定向劫持、CSRF | `ref/oauth-authentication-testing.md` |
| SAML 测试 | 签名绕过、断言篡改 | `ref/saml-authentication-testing.md` |
| 账户恢复测试 | 逻辑缺陷、枚举 | `ref/password-recovery-testing.md` |
| 账户枚举 | 用户存在性探测 | `ref/account-enumeration.md` |

#### 2.3.4 后测试阶段

- **结果验证** - 确认发现的漏洞
- **影响评估** - 评估业务影响
- **修复建议** - 提供具体修复方案
- **报告编写** - 编写详细测试报告

### 2.4 漏洞利用方法

#### 2.4.1 认证测试矩阵

```
                    低             中             高
影响程度          │──────────────│──────────────│──────────────│
                  │              │              │              │
 高               │ 账户枚举     │ 会话劫持     │ 认证绕过     │
                  │ 信息泄露     │ 权限提升     │ 完全接管     │
                  │              │              │              │
 中               │ 密码策略弱   │ MFA 绕过      │ 批量账户     │
                  │ 速率限制缺失  │ 令牌重放     │ 接管         │
                  │              │              │              │
 低               │ 错误消息     │ 日志缺失     │ 配置错误     │
                  │ 不够模糊     │ 审计困难     │ 信息泄露     │
                  │              │              │              │
                  │──────────────│──────────────│──────────────│
                    低             中             高
                        利用难度
```

#### 2.4.2 测试自动化的程度

| **测试类型** | **自动化程度** | **推荐工具** |
| :--- | :--- | :--- |
| 暴力破解 | 高 | Hydra、Burp Intruder |
| 账户枚举 | 高 | 自定义脚本 |
| 会话测试 | 中 | Burp Suite |
| JWT 测试 | 高 | jwt-tool |
| OAuth 测试 | 中 | Burp Suite、OAuth Tester |
| SAML 测试 | 中 | SAML Raider |
| MFA 测试 | 低 | 手动测试为主 |
| 逻辑缺陷 | 低 | 手动测试为主 |

### 2.5 测试报告框架

#### 2.5.1 报告结构

```
认证测试报告
├── 执行摘要
│   ├── 测试范围
│   ├── 风险概览
│   └── 关键发现
├── 测试方法
│   ├── 测试类型
│   ├── 测试工具
│   └── 测试时间
├── 详细发现
│   ├── 漏洞描述
│   ├── 复现步骤
│   ├── 影响评估
│   └── 修复建议
├── 附录
│   ├── 测试清单
│   ├── 工具列表
│   └── 参考资源
```

#### 2.5.2 风险评级标准

| **评级** | **标准** | **响应时间** |
| :--- | :--- | :--- |
| 严重 | 可直接导致账户接管或完全绕过认证 | 24 小时 |
| 高 | 需要一定条件但可导致未授权访问 | 7 天 |
| 中 | 需要多个条件组合或利用难度大 | 30 天 |
| 低 | 信息泄露或最佳实践偏离 | 90 天 |
