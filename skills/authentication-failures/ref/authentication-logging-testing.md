# **第二部分：核心渗透技术专题**

## 专题一：认证日志测试

### 2.1 技术介绍

认证日志测试是指对系统认证相关日志的记录、存储、分析和告警机制进行全面的安全评估。其本质是**验证系统能否正确记录认证事件，支持安全检测和事件响应**。

认证日志核心要素：

| **要素** | **描述** | **安全要求** |
| :--- | :--- | :--- |
| 登录事件 | 用户登录尝试记录 | 记录成功/失败 |
| 登出事件 | 用户登出记录 | 记录主动/被动 |
| 密码变更 | 密码修改记录 | 记录变更事件 |
| 权限变更 | 用户权限变更记录 | 记录变更详情 |
| 异常事件 | 异常登录行为 | 记录并告警 |
| 管理操作 | 管理员操作记录 | 详细记录操作 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **用户登录** | 登录尝试 | 未记录失败尝试 |
| **密码重置** | 忘记密码流程 | 未记录重置事件 |
| **权限变更** | 角色/权限修改 | 未记录变更详情 |
| **管理员操作** | 后台管理操作 | 未记录敏感操作 |
| **API 访问** | API 认证访问 | 未记录访问详情 |
| **批量操作** | 批量导入/导出 | 未记录操作内容 |
| **异常登录** | 异地/异常时间登录 | 未检测告警 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **日志完整性测试**
  - 执行各种认证操作
  - 检查是否有对应日志记录
  - 验证日志字段完整性
  - 测试日志时间准确性

- **日志注入测试**
  - 在用户名/密码中包含特殊字符
  - 尝试 CRLF 注入
  - 尝试日志伪造
  - 尝试日志溢出

- **日志绕过测试**
  - 尝试不触发日志的操作
  - 测试异常路径是否记录
  - 测试并发请求日志
  - 测试错误处理日志

- **告警机制测试**
  - 触发多次失败登录
  - 测试告警阈值
  - 测试告警通知
  - 测试告警内容

#### 2.3.2 白盒测试

- **代码审计**
  - 检查日志记录代码
  - 审计日志格式化逻辑
  - 检查日志级别配置
  - 查找日志绕过点

- **配置检查**
  - 检查日志配置文件
  - 审计日志存储配置
  - 检查日志保留策略
  - 检查日志访问控制

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

```bash
# 执行各种认证操作
# 1. 成功登录
# 2. 失败登录
# 3. 密码修改
# 4. 权限变更
# 然后检查日志系统

# 测试日志注入
curl -X POST https://target.com/login \
  -d "username=admin%0D%0AINFOMessage&password=test"

# 测试长用户名日志溢出
python3 -c "print('a'*10000)" | xargs -I {} \
  curl -X POST https://target.com/login -d "username={}&password=test"
```

#### 2.4.2 日志攻击技术

| **攻击类型** | **描述** | **影响** |
| :--- | :--- | :--- |
| 日志注入 | 在日志中注入虚假条目 | 混淆审计 |
| 日志伪造 | 伪造日志条目 | 嫁祸他人 |
| 日志溢出 | 超长输入导致日志截断 | 信息丢失 |
| 日志删除 | 删除或修改日志 | 销毁证据 |
| 日志绕过 | 不触发日志记录 | 隐藏行为 |

#### 2.4.3 日志注入攻击

```bash
# CRLF 注入
username=admin%0D%0A2024-01-01%2000:00:00%20INFOMessage%20Injected

# 日志伪造
# 如果日志格式不当，可能伪造条目
username="2024-01-01 00:00:00 SUCCESS admin from 127.0.0.1"

# 测试日志截断
# 使用超长用户名测试日志是否完整记录
```

#### 2.4.4 日志绕过

| **绕过方法** | **描述** | **检测方式** |
| :--- | :--- | :--- |
| 异常路径 | 通过异常路径访问 | 检查错误处理 |
| 并发请求 | 高频并发绕过 | 检查并发日志 |
| 特殊字符 | 特殊字符绕过记录 | 检查输入处理 |
| API 绕过 | 直接调用内部 API | 检查 API 日志 |

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过日志检测

- **低频操作** - 降低操作频率
- **正常行为模拟** - 模拟正常用户行为
- **分布式操作** - 使用多个账户/IP

#### 2.5.2 绕过告警机制

- **阈值测试** - 测试告警阈值
- **时间分散** - 延长操作间隔
- **多源分散** - 使用多个来源

#### 2.5.3 隐藏攻击痕迹

- **日志清理** - 如果有权限清理日志
- **日志覆盖** - 等待日志自动覆盖
- **日志混淆** - 制造大量噪声日志

## 3.3 修复建议

1. **完整记录** - 记录所有认证相关事件
2. **字段完整** - 包含时间、用户、IP、结果等字段
3. **时间同步** - 使用 NTP 同步时间
4. **日志保护** - 限制日志访问权限
5. **日志完整性** - 使用哈希保护日志
6. **集中存储** - 使用集中日志系统
7. **保留策略** - 设置合理的保留时间
8. **实时告警** - 对异常事件实时告警
9. **定期审计** - 定期审计日志内容
10. **防注入** - 正确转义日志内容
