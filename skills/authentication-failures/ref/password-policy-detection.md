# **第二部分：核心渗透技术专题**

## 专题一：密码策略检测

### 2.1 技术介绍

密码策略检测是指对系统密码复杂度要求、存储方式、更新策略等进行全面的安全评估。其本质是**评估系统是否实施了足够的密码安全控制，防止弱密码导致的账户泄露**。

密码策略核心要素：

| **要素** | **描述** | **安全要求** |
| :--- | :--- | :--- |
| 最小长度 | 密码最少字符数 | 至少 12 字符 |
| 复杂度要求 | 字符类型组合要求 | 大小写 + 数字 + 符号 |
| 密码历史 | 禁止重用旧密码 | 至少记住 10 个 |
| 最大有效期 | 密码最长使用时间 | 90 天或更短 |
| 最小有效期 | 密码最短使用时间 | 防止立即改回 |
| 锁定策略 | 失败尝试后锁定 | 5 次失败后锁定 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **用户注册** | 新用户设置密码 | 弱密码允许、无复杂度要求 |
| **密码修改** | 用户主动修改密码 | 允许弱密码、无历史检查 |
| **密码重置** | 忘记密码后重置 | 重置为弱密码、无复杂度要求 |
| **管理员创建用户** | 后台批量创建用户 | 默认弱密码、无强制修改 |
| **API 用户创建** | 通过 API 创建用户 | 无密码策略检查 |
| **批量导入用户** | CSV 导入用户 | 弱密码导入、无验证 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **密码复杂度测试**
  - 尝试设置短密码（如 1-5 字符）
  - 尝试设置纯数字密码
  - 尝试设置纯字母密码
  - 尝试设置常见弱密码（123456、password 等）
  - 尝试设置与用户名相同的密码

- **密码历史测试**
  - 记录当前密码
  - 修改为新密码
  - 尝试改回原密码
  - 重复多次测试历史记录

- **密码有效期测试**
  - 修改密码后立即尝试改回
  - 测试是否有最小有效期限制
  - 测试密码过期策略

- **锁定策略测试**
  - 连续输入错误密码
  - 记录锁定阈值
  - 测试锁定持续时间
  - 测试锁定后解锁方式

#### 2.3.2 白盒测试

- **代码审计**
  - 检查密码验证逻辑
  - 审计密码哈希算法
  - 检查密码策略配置
  - 查找硬编码密码

- **配置检查**
  - 检查密码策略配置文件
  - 审计密码哈希参数（如 bcrypt cost）
  - 检查密码存储方式
  - 检查日志记录配置

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

```bash
# 测试密码长度限制
for len in 1 2 3 4 5 6 7 8; do
  password=$(python3 -c "print('a'*$len)")
  curl -X POST https://target.com/register \
    -d "username=testuser&password=$password"
done

# 测试常见弱密码
weak_passwords=("123456" "password" "admin" "qwerty" "123456789")
for pwd in "${weak_passwords[@]}"; do
  curl -X POST https://target.com/register \
    -d "username=testuser&password=$pwd"
done
```

#### 2.4.2 密码策略绕过

| **绕过方法** | **描述** | **检测方式** |
| :--- | :--- | :--- |
| 前端绕过 | 禁用 JS 直接发送请求 | 比较前后端验证 |
| API 绕过 | 直接调用 API | 检查 API 验证 |
| 参数污染 | 发送多个 password 参数 | 检查参数处理 |
| 编码绕过 | 使用 Unicode 变体 | 检查字符处理 |
| 空格填充 | 在密码前后加空格 | 检查 trim 处理 |

#### 2.4.3 弱密码利用

```bash
# 使用已知弱密码尝试登录
weak_passwords=("123456" "password" "admin" "qwerty" "123456789" "12345678")
for pwd in "${weak_passwords[@]}"; do
  curl -X POST https://target.com/login \
    -d "username=admin&password=$pwd"
done

# 使用用户名作为密码
curl -X POST https://target.com/login \
  -d "username=admin&password=admin"
```

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过复杂度检查

- **前端禁用** - 禁用 JavaScript 绕过前端验证
- **直接 API** - 绕过前端直接调用 API
- **Unicode 绕过** - 使用特殊 Unicode 字符
- **大小写绕过** - 如果仅检查大小写存在性

#### 2.5.2 绕过历史记录

- **循环修改** - 在允许的范围内循环修改
- **大小写变换** - 改变大小写绕过哈希匹配
- **添加后缀** - 添加数字或符号后缀

#### 2.5.3 绕过锁定策略

- **IP 轮换** - 使用代理池分散尝试
- **用户名变异** - 尝试不同用户名
- **时间分散** - 延长尝试间隔
- **密码喷洒** - 对多用户尝试少量密码

# **第三部分：附录**

## 3.1 密码策略检测检查清单

| **检查项** | **测试方法** | **预期结果** |
| :--- | :--- | :--- |
| 最小长度 | 尝试短密码 | 应拒绝少于 12 字符 |
| 复杂度要求 | 尝试简单密码 | 应拒绝弱密码 |
| 常见密码检查 | 尝试 123456 等 | 应拒绝常见密码 |
| 用户名检查 | 使用用户名作密码 | 应拒绝 |
| 密码历史 | 尝试重用旧密码 | 应拒绝最近 10 个 |
| 最大有效期 | 检查密码过期策略 | 应不超过 90 天 |
| 锁定策略 | 多次失败登录 | 应锁定账户 |
| 哈希算法 | 检查存储方式 | 应使用 bcrypt/Argon2 |

## 3.2 密码策略测试工具

| **工具名称** | **用途** | **命令示例** |
| :--- | :--- | :--- |
| **Burp Suite** | 拦截/修改请求 | 手动测试密码策略 |
| **Hydra** | 暴力破解 | `hydra -l user -P rockyou.txt` |
| **hashcat** | 密码破解 | `hashcat -m 0 hashes.txt` |
| **john** | 密码破解 | `john --wordlist=rockyou.txt hashes` |
| **自定义脚本** | 自动化测试 | Python 编写策略测试 |

## 3.3 修复建议

1. **最小长度** - 要求至少 12 字符
2. **复杂度要求** - 要求大小写 + 数字 + 符号组合
3. **常见密码检查** - 拒绝常见弱密码列表
4. **用户名检查** - 拒绝包含用户名的密码
5. **密码历史** - 记住最近 10 个密码
6. **最大有效期** - 90 天或更短
7. **最小有效期** - 防止立即改回（如 24 小时）
8. **锁定策略** - 5 次失败后锁定 15-30 分钟
9. **安全哈希** - 使用 bcrypt 或 Argon2
10. **密码管理器** - 鼓励使用密码管理器
