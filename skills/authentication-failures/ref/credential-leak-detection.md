# **第二部分：核心渗透技术专题**

## 专题一：凭证泄露检测

### 2.1 技术介绍

凭证泄露检测是指对企业相关用户名和密码组合在各种渠道的泄露情况进行监测和评估。其本质是**发现已被泄露的凭证，防止攻击者利用这些凭证进行未授权访问**。

凭证泄露常见渠道：

| **渠道** | **描述** | **风险等级** |
| :--- | :--- | :--- |
| 数据泄露 | 第三方服务数据泄露 | 高 |
| 代码仓库 | GitHub 等公开代码库 | 高 |
| 暗网市场 | 暗网凭证交易市场 | 高 |
|  Paste 站点 | Pastebin 等文本分享 | 中 |
| 钓鱼攻击 | 钓鱼网站收集凭证 | 高 |
| 恶意软件 | 键盘记录、浏览器窃取 | 高 |
| 内部泄露 | 内部人员故意/无意泄露 | 中 - 高 |

### 2.2 检测常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **员工邮箱泄露** | 企业邮箱凭证泄露 | 邮箱接管、钓鱼 |
| **云服务凭证** | AWS、Azure 密钥泄露 | 云资源未授权访问 |
| **数据库凭证** | 数据库连接字符串泄露 | 数据泄露 |
| **API 密钥** | 第三方 API Key 泄露 | API 滥用 |
| **管理员凭证** | 系统管理员凭证泄露 | 系统接管 |
| **服务账户** | 自动化服务凭证泄露 | 服务滥用 |
| **代码中的硬编码** | 源码中的凭证泄露 | 直接利用 |

### 2.3 漏洞探测方法

#### 2.3.1 外部泄露检测

- **公开数据源检测**
  - 使用 Have I Been Pwned 检查邮箱泄露
  - 使用 DeHashed 等工具搜索泄露数据
  - 监控暗网市场和企业相关关键词
  - 监控 Paste 站点敏感信息

- **代码仓库检测**
  - 扫描 GitHub 等公开代码仓库
  - 使用 GitGuardian 等工具监控
  - 检查企业相关代码中的凭证
  - 监控 Fork 和 Gist 中的敏感信息

- **域名相关检测**
  - 监控企业域名相关的泄露
  - 检查子域名相关的凭证
  - 监控相似域名的钓鱼网站

#### 2.3.2 内部泄露检测

- **日志分析**
  - 检查应用日志中的凭证
  - 检查调试日志中的敏感信息
  - 检查错误消息中的凭证泄露
  - 检查版本控制系统历史

- **代码审计**
  - 扫描代码库中的硬编码凭证
  - 检查配置文件中的凭证
  - 检查环境变量中的凭证
  - 检查文档中的凭证

- **网络流量分析**
  - 监控明文传输的凭证
  - 检测异常的凭证使用
  - 监控凭证的批量访问

#### 2.3.3 主动检测

- **蜜罐凭证**
  - 在代码中放置蜜罐凭证
  - 监控蜜罐凭证的使用
  - 追踪凭证泄露源头

- **凭证监控**
  - 定期检查关键账户凭证
  - 监控凭证使用模式
  - 检测异常的登录行为

### 2.4 漏洞利用方法

#### 2.4.1 基础信息收集

```bash
# 使用 Have I Been Pwned API
curl https://haveibeenpwned.com/api/v3/breachedaccount/user@example.com

# 使用 GitHub 搜索
# 搜索企业相关的凭证
site:github.com "company.com" password
site:github.com "company.com" api_key
site:github.com "company.com" secret

# 使用 Google Dork
site:pastebin.com "company.com" password
site:pastebin.com "@company.com" credential
```

#### 2.4.2 凭证利用测试

| **操作** | **方法** | **目的** |
| :--- | :--- | :--- |
| 凭证验证 | 使用泄露凭证尝试登录 | 验证凭证有效性 |
| 凭证填充 | 批量尝试泄露凭证 | 评估影响范围 |
| 权限评估 | 检查可访问的资源 | 评估泄露影响 |
| 横向移动 | 使用凭证访问其他系统 | 评估关联风险 |

#### 2.4.3 凭证填充攻击

```bash
# 使用凭证填充工具
python3 credstuff.py -l leaked_credentials.txt \
  -t https://target.com/login \
  -u USERNAME -p PASSWORD

# 使用 Snipr 等工具
snipr -f credentials.txt -t target.com
```

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过速率限制

- **IP 轮换** - 使用代理池分散请求
- **时间分散** - 延长请求间隔
- **多账户轮换** - 在多个账户间轮换

#### 2.5.2 绕过检测机制

- **低频尝试** - 降低尝试频率
- **正常行为模拟** - 模拟正常用户行为
- **地理分散** - 使用不同地理位置的 IP

#### 2.5.3 凭证更新绕过

- **旧凭证测试** - 测试历史凭证
- **密码模式分析** - 分析用户密码模式
- **变体尝试** - 尝试常见密码变体

## 3.3 修复建议

1. **立即重置** - 对已泄露凭证立即重置
2. **强制修改** - 要求用户修改密码
3. **启用 MFA** - 对所有账户启用多因素认证
4. **凭证管理** - 使用密码管理器
5. **代码扫描** - 定期扫描代码中的凭证
6. **日志清理** - 清理日志中的敏感信息
7. **员工培训** - 提高员工安全意识
8. **持续监控** - 建立持续监控机制
9. **事件响应** - 制定凭证泄露响应流程
10. **最小权限** - 实施最小权限原则
