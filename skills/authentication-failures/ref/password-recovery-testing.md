# **第二部分：核心渗透技术专题**

## 专题一：密码恢复测试

### 2.1 技术介绍

密码恢复测试是指对密码重置和账户恢复流程进行全面的安全评估。其本质是**验证密码恢复机制能否安全地验证用户身份，防止账户接管和未授权密码重置**。

密码恢复常见类型：

| **类型** | **描述** | **安全风险** |
| :--- | :--- | :--- |
| 邮件重置 | 发送重置链接到邮箱 | 邮箱接管、链接泄露 |
| 短信重置 | 发送验证码到手机 | SIM 交换、拦截 |
| 安全问题 | 回答预设的安全问题 | 信息可猜测、社会工程 |
| 备用邮箱 | 发送到备用邮箱 | 备用邮箱泄露 |
| 人工审核 | 客服人工验证 | 社会工程、内部威胁 |
| 多因素恢复 | 多种验证组合 | 实现复杂度 |

### 2.2 测试常见于哪些业务场景

| **业务场景** | **功能示例** | **风险点描述** |
| :--- | :--- | :--- |
| **忘记密码** | "忘记密码"功能 | 重置链接/验证码泄露 |
| **账户找回** | 无法访问邮箱时的找回 | 安全问题可猜测 |
| **邮箱修改** | 修改绑定邮箱 | 未验证原邮箱 |
| **手机修改** | 修改绑定手机 | 未验证原手机 |
| **客服重置** | 联系客服重置密码 | 社会工程攻击 |
| **批量重置** | 管理员批量重置密码 | 默认密码、批量泄露 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

- **账户枚举测试**
  - 输入不同邮箱测试响应差异
  - 分析响应时间差异
  - 分析错误消息差异
  - 测试邮箱存在性泄露

- **重置令牌测试**
  - 分析重置令牌的格式和熵值
  - 测试令牌有效期
  - 测试令牌一次性使用
  - 测试令牌与用户绑定
  - 测试令牌重用

- **流程绕过测试**
  - 尝试跳过中间步骤
  - 尝试修改请求参数
  - 尝试直接访问重置页面
  - 尝试修改用户标识参数

- **安全问题测试**
  - 测试安全问题答案可猜测性
  - 测试答案大小写敏感
  - 测试答案格式验证
  - 尝试暴力破解答案

#### 2.3.2 白盒测试

- **代码审计**
  - 检查重置令牌生成逻辑
  - 审计令牌验证代码
  - 检查速率限制实现
  - 查找硬编码绕过

- **配置检查**
  - 检查令牌有效期配置
  - 检查速率限制配置
  - 检查通知配置
  - 检查日志记录

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过速率限制

- **IP 轮换** - 使用代理池分散请求
- **邮箱变异** - 使用 `user+1@example.com` 等
- **时间分散** - 延长请求间隔

#### 2.5.2 绕过通知机制

- **邮箱规则利用** - 利用邮箱别名规则
- **短信转发** - 设置短信转发
- **邮箱过滤器** - 创建邮件过滤规则

## 3.1 密码恢复测试检查清单

| **检查项** | **测试方法** | **预期结果** |
| :--- | :--- | :--- |
| 账户枚举 | 输入不同邮箱 | 响应应相同 |

| 令牌一次性 | 重复使用令牌 | 应拒绝重用 |
| 用户绑定 | 交叉使用令牌 | 应拒绝 |

| 安全问题 | 尝试常见答案 | 应有足够复杂度 |

## 3.3 修复建议

1. **统一响应** - 不区分邮箱是否存在
2. **强令牌** - 使用高熵随机令牌
3. **短期有效** - 令牌有效期不超过 15 分钟
4. **一次性使用** - 使用后立即失效
5. **用户绑定** - 令牌与用户严格绑定
6. **速率限制** - 限制重置请求频率
7. **多因素验证** - 要求额外验证因素
8. **安全通知** - 通知用户密码重置事件
9. **审计日志** - 记录所有重置事件
10. **避免安全问题** - 使用更安全的恢复方式
