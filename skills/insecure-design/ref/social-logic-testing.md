# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 社交业务逻辑漏洞原理

社交业务逻辑漏洞是指社交网络平台在用户关系、内容传播、互动机制等业务流程设计中的安全缺陷。

**核心业务环节**：
- 用户注册与认证
- 好友关系管理
- 内容发布与传播
- 消息通讯
- 点赞评论互动
- 群组管理

**本质问题**：
- 用户关系验证缺陷
- 内容访问控制不足
- 互动机制设计漏洞
- 隐私设置设计缺陷

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-840 | 业务逻辑缺陷 |
| CWE-200 | 信息泄露 |
| CWE-284 | 访问控制不当 |

## 2.2 攻击常见于哪些业务场景

| 业务场景 | 功能示例 | 风险点描述 |
|---------|---------|-----------|
| 好友关系 | 添加/删除好友 | 批量添加、关系伪造 |
| 隐私设置 | 可见性控制 | 隐私绕过、信息泄露 |
| 内容发布 | 动态/帖子发布 | 内容篡改、越权删除 |
| 消息通讯 | 私聊/群聊 | 消息窃取、冒充发送 |
| 点赞评论 | 互动功能 | 刷赞、恶意评论 |
| 关注系统 | 关注/粉丝 | 批量关注、僵尸粉 |
| 群组管理 | 创建/加入群组 | 越权管理、恶意入群 |
| 举报系统 | 内容举报 | 恶意举报、举报绕过 |

## 2.3 漏洞发现方法

### 2.3.1 业务流程分析

**步骤 1：绘制用户关系图**

```
典型社交关系：
用户 → 好友（双向）
用户 → 关注（单向）
用户 → 群组成员
用户 → 黑名单
```

**步骤 2：识别内容访问控制**

```
内容可见性：
- 公开（所有人可见）
- 好友可见
- 仅自己可见
- 指定人可见
- 指定人不可见
```

**步骤 3：识别信任边界**

```
客户端 vs 服务端：
- 好友关系验证（应在服务端）
- 内容访问控制（应在服务端）
- 隐私设置（应在服务端）
- 消息加密（端到端）
```

### 2.3.2 关键参数识别

```
社交请求中的关键参数：
- 用户 ID/目标用户 ID
- 内容 ID/帖子 ID
- 关系类型（好友/关注/屏蔽）
- 可见性设置
- 群组 ID
- 消息 ID
```

## 2.4 漏洞测试方法

### 2.4.1 好友关系绕过测试

```bash
# 场景：好友可见内容

# 1. 正常好友请求
POST /api/friend/add
{"target_user_id": "user123"}

# 2. 非好友访问好友内容
GET /api/post/456  # 好友可见内容
# 检查是否验证好友关系

# 3. 删除好友后访问
# 删除好友后尝试访问历史内容

# 4. 待审核状态访问
# 在好友请求待审核时访问
```

### 2.4.2 隐私设置绕过测试

```bash
# 场景：隐私内容访问

# 1. 遍历 ID 访问私密内容
GET /api/post/1
GET /api/post/2
GET /api/post/3
# 检查是否验证权限

# 2. 修改可见性参数
POST /api/post/create
{
    "content": "test",
    "visibility": "public"  # 强制公开
}

# 3. 通过关联资源访问
# 通过评论、分享等间接访问
```

### 2.4.3 内容越权测试

```bash
# 场景：内容管理

# 1. 越权删除
DELETE /api/post/999  # 他人内容
# 检查是否验证所有权

# 2. 越权编辑
PUT /api/post/999
{"content": "modified"}

# 3. 越权审核
POST /api/content/approve
{"content_id": 999, "status": "approved"}
```

### 2.4.4 消息窃取测试

```bash
# 场景：私信功能

# 1. 遍历消息 ID

# 检查是否验证收发件人

# 2. 修改收件人

    "content": "test"
}

# 3. 群聊消息越权

```

### 2.4.5 互动功能滥用测试

```bash

# 1. 批量点赞

    POST /api/like {"post_id": $post_id}

# 多次提交相同内容

# 3. 自问自答
# 同一账户提问和回答刷分
```

### 2.4.6 关注系统滥用测试

```bash

# 1. 批量关注

    POST /api/follow {"user_id": $user_id}

# 2. 关注计数篡改

{"user_id": "target", "count": 9999}

# 3. 僵尸粉检测绕过

```

### 2.4.7 群组管理测试

```bash
# 场景：群组功能

# 1. 越权加入群组
POST /api/group/join
{"group_id": "private_group"}
# 私有群组检查

# 2. 越权管理

{
    "group_id": "group1",

{
    "group_id": "group1",
    "is_public": true  # 私有改公开
```

## 2.5 漏洞利用方法

### 2.5.1 信息收集攻击

```bash
# 利用隐私绕过收集信息

# 1. 遍历用户资料
# 收集公开和半公开信息

# 2. 通过好友列表扩展
# 利用好友关系网络

# 3. 通过内容关联
# 从评论、点赞推断关系
```

### 2.5.2 社交工程攻击

```bash
# 利用平台功能进行社工

# 1. 冒充熟人
# 使用相似头像和名称

# 2. 虚假身份
# 创建多个虚假人设

# 3. 信任链攻击
# 通过好友的好友建立信任
```

### 2.5.3 影响力操纵

```bash
# 操纵平台影响力指标

# 1. 刷粉丝
# 批量关注制造假象

# 2. 刷互动
# 批量点赞评论

# 3. 舆论操纵
# 多账户发布相同内容
```

# 第三部分：附录

## 3.1 社交业务逻辑测试检查清单

```
□ 好友关系是否验证
□ 隐私设置是否有效
□ 内容访问是否授权
□ 消息是否越权访问
□ 互动功能是否限流
□ 群组权限是否验证
□ 举报是否有限制
□ 身份是否可冒充
□ 是否有反垃圾机制
□ 是否有异常行为检测
```

## 3.3 修复建议

1. **关系验证** - 严格验证用户关系
2. **访问控制** - 基于角色的内容访问
3. **隐私保护** - 服务端执行隐私设置
4. **速率限制** - 互动功能限流
5. **内容审核** - 自动 + 人工审核
6. **异常检测** - 检测异常行为模式
7. **身份验证** - 强化身份认证
8. **审计日志** - 记录关键操作
