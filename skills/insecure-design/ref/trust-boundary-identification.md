# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 信任边界原理

信任边界是指系统中不同信任级别区域之间的分界线。跨越信任边界的数据或操作需要经过严格的安全验证。

**核心概念**：
- 信任级别（Trust Level）
- 边界跨越（Boundary Crossing）
- 数据验证（Data Validation）
- 身份传递（Identity Propagation）

**本质问题**：
- 过度信任导致漏洞
- 边界验证不足
- 信任传递缺陷

### 常见信任边界类型

| 边界类型 | 描述 | 风险等级 |
|---------|------|---------|
| 用户 ↔ 应用 | 外部用户与系统边界 | 高 |
| 应用 ↔ 数据库 | 应用层与数据层边界 | 中 |
| 内部服务 ↔ 内部服务 | 微服务间边界 | 中 - 高 |
| 应用 ↔ 第三方 | 与外部服务边界 | 高 |
| 网络区域 ↔ 网络区域 | 网络分段边界 | 中 |

## 2.2 信任边界识别

### 2.2.1 系统分层分析

**典型三层架构信任边界**

```
┌─────────────────────────────────┐
│         客户端层                 │  ◄─── 不信任
│  (Browser/Mobile/Desktop)       │
├═════════════════════════════════┤  ◄─── 信任边界 1
│         Web 应用层               │  ◄─── 部分信任
│  (Web Server/Application)       │
├═════════════════════════════════┤  ◄─── 信任边界 2
│         数据层                   │  ◄─── 相对信任
│  (Database/Cache/Storage)       │
└─────────────────────────────────┘
```

**微服务架构信任边界**

```
┌─────────────────────────────────┐
│         API Gateway              │  ◄─── 外部信任边界
├═════════════════════════════════┤
│         服务 A    服务 B          │  ◄─── 内部信任边界
│                    服务 C         │
├═════════════════════════════════┤
│      数据库    消息队列          │
└─────────────────────────────────┘
```

### 2.2.2 数据流信任分析

**步骤 1：追踪数据路径**

```
用户输入数据路径：
用户 → 浏览器 → 网络 → 负载均衡 → 
Web 服务器 → 应用服务器 → 数据库
```

**步骤 2：识别边界跨越**

```
每次数据跨越：
- 网络边界
- 进程边界
- 权限边界
- 信任域边界
```

**步骤 3：评估验证措施**

```
每个边界检查：
- 输入验证
- 身份验证
- 授权检查
- 数据加密
```

### 2.2.3 代码级信任边界

**函数/方法边界**

```python
# 信任边界示例

# ❌ 错误的信任假设
def process_user_input(user_input):
    # 直接使用用户输入（假设已验证）
    query = f"SELECT * FROM users WHERE id = {user_input}"
    return execute(query)

# ✅ 正确的信任处理
def process_user_input(user_input):
    # 不信任输入，进行验证
    if not validate_integer(user_input):
        raise ValueError("Invalid input")
    # 使用参数化查询
    return db.query("SELECT * FROM users WHERE id = ?", user_input)
```

**模块/组件边界**

```
模块间调用：
- 不假设输入可信
- 验证所有参数
- 记录跨模块调用
- 处理异常情况
```

## 2.4 常见信任边界漏洞

### 2.4.1 客户端信任

```
漏洞模式：
- 信任客户端验证
- 信任隐藏字段
- 信任 Cookie 值
- 信任请求头

测试要点：
- 绕过客户端验证
- 修改隐藏字段
- 篡改 Cookie
- 伪造请求头
```

### 2.4.2 内部网络信任

```
漏洞模式：
- 内网服务无认证
- SSRF 利用内网信任
- 内网横向移动

测试要点：
- 扫描内网服务
- 测试 SSRF
- 尝试横向移动
```

### 2.4.3 第三方信任

```
漏洞模式：
- 过度信任第三方数据
- 第三方回调未验证
- 供应链攻击

测试要点：
- 伪造第三方回调
- 注入恶意数据
- 测试依赖链
```

### 2.4.4 缓存信任

```
漏洞模式：
- 缓存投毒
- 缓存跨越用户
- 缓存权限绕过

测试要点：
- 测试缓存键碰撞
- 测试跨用户缓存
- 测试缓存注入
```

## 3.3 最佳实践

1. **零信任原则** - 不信任任何输入
2. **边界明确** - 清晰定义信任边界
3. **验证所有跨越** - 边界跨越必须验证
4. **最小信任** - 授予最小必要信任
5. **文档化** - 记录所有信任假设
6. **持续审查** - 定期审查信任边界
