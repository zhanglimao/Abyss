# 第一部分：文档概述

## 1.1 编写目的

## 1.2 适用范围

本文档适用于所有具有角色权限管理功能的 Web 应用、企业系统和 API 服务。

本文件主要面向执行渗透测试任务的安全工程师、安全分析师。

# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

权限设计攻击是指攻击者利用应用程序权限模型设计中的缺陷，获取超出其应有范围的访问权限或执行未授权操作。

**本质问题**：
- 角色权限分配不合理
- 权限继承设计缺陷
- 默认权限配置不当
- 权限检查逻辑缺失

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-269 | 权限提升 |
| CWE-275 | 权限检查缺失 |
| CWE-284 | 访问控制不当 |

## 2.2 攻击常见于哪些业务场景

| 业务场景 | 功能示例 | 风险点描述 |
|---------|---------|-----------|
| 用户管理 | 角色分配、权限修改 | 自我提权、越权分配 |
| 后台管理 | 管理员功能 | 未授权访问管理功能 |
| 数据访问 | 数据查看/编辑 | 越权访问他人数据 |
| API 接口 | 接口调用权限 | 未授权调用敏感接口 |
| 文件管理 | 文件上传/下载 | 越权访问他人文件 |
| 审批流程 | 审批权限 | 越权审批、自审批 |
| 配置管理 | 系统配置修改 | 未授权修改配置 |
| 审计日志 | 日志查看/删除 | 越权查看/删除日志 |

## 2.4 漏洞利用方法

### 2.4.1 角色参数篡改

```bash

{
    "username": "attacker",
    "password": "pass123",
    "role": "user"
}

# 2. 篡改角色
POST /api/register
{
    "username": "attacker",
    "password": "pass123",
    "role": "admin"  # 直接修改为管理员
}

# 3. 或者添加隐藏角色

    "password": "pass123",
```

```bash
# 满足任一条件即可获得权限
```

# 第三部分：附录

## 3.1 权限设计测试检查清单

```
□ 角色是否可被用户修改
□ 权限检查是否在服务端
□ 是否有默认过度授权
□ 权限继承是否安全
□ 权限缓存是否及时更新
□ 是否有最小权限原则
□ 权限是否可组合滥用
□ 是否有权限审计日志
□ 权限检查是否一致
□ 是否有上下文权限控制
```

## 3.3 修复建议

1. **服务端权限检查** - 所有权限检查在服务端进行
2. **最小权限原则** - 默认授予最小必要权限
3. **角色不可篡改** - 用户不可自行修改角色
4. **权限缓存同步** - 确保权限变更及时生效
5. **权限审计** - 记录所有权限相关操作
6. **统一权限检查** - 使用统一的权限检查框架
7. **定期权限审查** - 定期审查权限分配合理性
