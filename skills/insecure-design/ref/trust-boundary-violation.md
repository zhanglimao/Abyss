# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

信任边界违规是指攻击者利用系统对不可信数据的过度信任，将恶意数据传递到信任区域，从而执行未授权操作。

**信任边界定义**：
- 客户端（不可信）↔ 服务端（可信）
- 外部 API（不可信）↔ 内部系统（可信）
- 用户输入（不可信）↔ 业务逻辑（可信）

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-501 | 信任边界违规 |
| CWE-20 | 不当输入验证 |

## 2.2 攻击常见于哪些业务场景

| 场景 | 信任边界 | 风险点 |
|-----|---------|-------|
| 前端→后端 | HTTP 请求 | 参数未验证 |
| 微服务间 | API 调用 | 内部 API 无认证 |
| 第三方集成 | Webhook | 回调未验证 |
| 缓存系统 | Cache 数据 | 缓存投毒 |
| 日志系统 | 日志内容 | 日志注入 |

## 2.3 漏洞发现方法

### 2.3.1 信任边界识别

```
识别系统中的信任边界：

1. 数据流分析
   用户输入 → 输入验证 → 业务逻辑 → 数据存储
   
2. 组件边界分析
   浏览器 ↔ Web 服务器 ↔ 应用服务器 ↔ 数据库
   
3. 网络边界分析
   互联网 ↔ DMZ ↔ 内网 ↔ 核心网络
```

### 2.3.2 边界穿越测试

```bash
# 测试数据从不可信区域到可信区域的传递

# 1. 直接发送恶意数据到服务端
POST /api/data
{"trusted_field": "malicious_value"}

# 2. 通过中间组件传递
# 用户输入 → 缓存 → 服务端使用

# 3. 通过第三方系统传递
# 用户输入 → Webhook → 内部系统
```

### 2.3.3 隐式信任检测

```bash
# 检测服务端是否隐式信任某些来源

# HTTP 头信任
X-Forwarded-For: 127.0.0.1
X-Real-IP: 127.0.0.1

# Cookie 信任
is_admin=true
role=superuser

# 内部头信任
X-Internal-Request: true
X-From-Service: trusted-service
```

## 2.4 漏洞利用方法

### 2.4.1 HTTP 头注入

```bash
# 如果服务端信任某些 HTTP 头
curl -H "X-Forwarded-For: 127.0.0.1" \
     -H "X-Real-IP: 127.0.0.1" \
     https://target.com/admin

# 绕过 IP 限制
```

### 2.4.2 缓存投毒

```bash
# 如果服务端信任缓存数据
# 1. 向缓存注入恶意数据
# 2. 服务端从缓存读取并使用
# 3. 恶意数据被当作可信数据处理

# 示例：缓存用户会话
# 攻击者设置恶意会话数据
# 服务端从缓存读取后信任
```

### 2.4.3 内部 API 冒充

```bash
# 如果内部 API 无认证
# 从外部直接调用

curl -H "X-From-Service: payment-service" \
     https://target.com/internal/api/transfer

# 绕过服务间认证
```

### 2.4.4 Webhook 伪造

```bash
# 如果 Webhook 无签名验证
# 伪造第三方回调

POST /webhook/payment
{
    "event": "payment.completed",
    "order_id": "12345",
    "amount": 999.99
}

# 伪造支付完成通知
```

### 2.4.5 日志信任攻击

```bash
# 如果系统信任日志中的内容
# 注入恶意日志条目

# 攻击者触发日志记录
# 日志中包含恶意数据
# 日志分析系统处理恶意数据
# 执行非预期操作
```

## 2.5 漏洞利用绕过方法

### 2.5.1 验证逻辑绕过

```bash
# 如果验证逻辑有缺陷
# 尝试：
# - 不同参数名
# - 不同数据类型
# - 不同编码方式
```

### 2.5.2 多层边界穿越

```
穿越多个信任边界：
用户输入 → CDN → WAF → 负载均衡 → 应用 → 数据库

每一层都可能引入信任假设
```

# 第三部分：附录

## 3.1 信任边界测试检查清单

```
□ 识别所有信任边界
□ 测试边界验证逻辑
□ 测试 HTTP 头信任
□ 测试内部 API 认证
□ 测试缓存数据验证
□ 测试 Webhook 签名
□ 测试日志数据处理
□ 测试第三方集成
```

## 3.2 常用工具

| 工具 | 用途 | 链接 |
|-----|------|------|
| Burp Suite | 请求篡改 | https://portswigger.net/burp |
| OWASP ZAP | 自动化扫描 | https://www.zaproxy.org/ |
| 数据流分析工具 | 识别信任边界 | 各种 SAST 工具 |

## 3.3 修复建议

1. **明确信任边界** - 文档化所有信任边界
2. **边界验证** - 所有跨边界数据必须验证
3. **零信任架构** - 不隐式信任任何来源
4. **深度防御** - 多层验证机制
