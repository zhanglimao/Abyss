# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 数据流图原理

数据流图（Data Flow Diagram, DFD）是一种图形化表示系统数据流动、存储和处理的方法，是威胁建模的基础工具。

**核心元素**：
- 外部实体（External Entity）
- 过程（Process）
- 数据存储（Data Store）
- 数据流（Data Flow）

**本质作用**：
- 可视化数据流动
- 识别信任边界
- 发现攻击面
- 指导安全测试

### DFD 符号说明

```
符号表示：
┌─────────┐
│  矩形   │  外部实体（用户、外部系统）
├─────────┤
│  圆形   │  过程（处理逻辑）
│  或圆角矩形 │
├═════════┤
│  双横线  │  数据存储（数据库、文件）
├─────────┤
│  →      │  数据流（数据传递方向）
```

## 2.2 数据流图绘制

### 2.2.1 上下文图（Level 0 DFD）

**步骤 1：识别系统边界**

```
系统边界定义：
- 系统内部组件
- 系统外部实体
- 输入/输出接口
```

**步骤 2：绘制上下文图**

```
示例：电商系统上下文图

     ┌──────────┐
     │  客户    │
     └────┬─────┘
          │ 订单请求
          ▼
     ┌───────────────┐
     │               │
     │   电商系统    │
     │               │
     └───────┬───────┘
             │ 订单确认
     ┌───────▼───────┐
     │  支付网关     │
     └───────────────┘
```

### 2.2.2 详细图（Level 1 DFD）

**步骤 1：分解主要过程**

```
电商系统主要过程：
1. 用户认证
2. 商品浏览
3. 购物车管理
4. 订单处理
5. 支付处理
6. 库存管理
```

**步骤 2：绘制 Level 1 DFD**

```
示例：订单处理 DFD

     ┌────────┐
     │ 客户   │
     └───┬────┘
         │ 订单请求
         ▼
     ┌───────────┐
     │ 验证订单  │
     └─────┬─────┘
           │ 有效订单
           ▼
     ┌───────────┐     ┌──────────┐
     │ 创建订单  │────▶│ 订单数据库│
     └─────┬─────┘     └──────────┘
           │ 订单确认
           ▼
     ┌───────────┐
     │ 通知客户  │
     └───────────┘
```

### 2.2.3 信任边界标注

**步骤 1：识别信任边界**

```
常见信任边界：
- 用户 ↔ Web 应用（不信任）
- Web 应用 ↔ API 服务（部分信任）
- API 服务 ↔ 数据库（部分信任）
- 内部服务 ↔ 内部服务（可能过度信任）
```

**步骤 2：标注信任边界**

```
示例：

     ┌────────┐
     │ 用户   │  ◄─── 不信任区域
     └───┬────┘
═══════════════════════  ◄─── 信任边界 1
         │
     ┌───▼────┐
     │ Web 层 │  ◄─── 部分信任区域
     └───┬────┘
═══════════════════════  ◄─── 信任边界 2
         │
     ┌───▼────┐
     │ API 层 │  ◄─── 部分信任区域
     └───┬────┘
═══════════════════════  ◄─── 信任边界 3
         │
     ┌───▼────┐
     │ 数据库 │  ◄─── 信任区域
     └────────┘
```

## 2.3 数据流图分析

### 2.3.1 攻击面识别

**外部实体分析**

```
分析要点：
- 哪些实体是不可信的？
- 实体输入是否验证？
- 实体认证是否充分？
```

**数据流分析**

```
分析要点：
- 数据流是否加密？
- 数据流是否可篡改？
- 数据流是否可重放？
```

**数据存储分析**

```
分析要点：
- 存储数据是否加密？
- 访问控制是否充分？
- 审计日志是否完整？
```

**过程分析**

```
分析要点：
- 过程是否有输入验证？
- 过程是否有权限检查？
- 过程是否有审计日志？
```

### 2.3.2 STRIDE 威胁分析

**针对每个 DFD 元素应用 STRIDE**

```
外部实体威胁：
- 欺骗：攻击者冒充合法实体
- 篡改：实体发送恶意数据

数据流威胁：
- 篡改：数据在传输中被修改
- 信息泄露：数据被窃听
- 拒绝服务：流量淹没

数据存储威胁：
- 篡改：数据被修改
- 信息泄露：数据被读取
- 权限提升：越权访问

过程威胁：
- 所有 STRIDE 类型都可能
```

### 2.3.3 攻击路径分析

**步骤 1：识别入口点**

```
常见入口点：
- Web 表单
- API 端点
- 文件上传
- 网络接口
```

**步骤 2：识别目标点**

```
常见目标点：
- 敏感数据存储
- 管理功能
- 核心业务逻辑
- 第三方集成
```

**步骤 3：绘制攻击路径**

```
示例攻击路径：

用户输入 → Web 表单 → 输入验证（绕过）→ 
业务逻辑 → 数据库查询（注入）→ 数据泄露
```

## 2.4 安全测试用例生成

### 2.4.1 基于 DFD 的测试用例

```
针对每个数据流：
- 测试数据完整性
- 测试数据机密性
- 测试数据可用性

针对每个数据存储：
- 测试访问控制
- 测试数据加密
- 测试审计日志

针对每个过程：
- 测试输入验证
- 测试输出编码
- 测试错误处理

针对每个外部实体：
- 测试认证机制
- 测试授权机制
- 测试速率限制
```

### 2.4.2 测试用例示例

```
用例 1：用户认证数据流测试
- 测试明文密码传输
- 测试暴力破解
- 测试会话劫持

用例 2：订单数据存储测试
- 测试 SQL 注入
- 测试越权访问
- 测试数据篡改

用例 3：支付处理过程测试
- 测试金额篡改
- 测试重放攻击
- 测试业务逻辑绕过
```

# 第三部分：附录

## 3.1 数据流图分析检查清单

```
□ 系统边界是否明确
□ 外部实体是否识别完整
□ 数据流是否标注清晰
□ 数据存储是否识别
□ 处理过程是否分解
□ 信任边界是否标注
□ 攻击面是否识别
□ 威胁分析是否完成
□ 测试用例是否生成
□ 文档是否更新
```

## 3.3 最佳实践

1. **分层绘制** - 从上下文图到详细图
2. **迭代更新** - 系统变更时更新 DFD
3. **团队评审** - 多角色评审 DFD
4. **标注清晰** - 明确标注信任边界
5. **关联威胁** - 将 DFD 与威胁分析关联
6. **生成用例** - 从 DFD 生成测试用例
