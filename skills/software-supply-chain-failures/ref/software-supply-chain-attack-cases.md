### 2.1.4 核心技术："花生酱杯剃须刀片"技术

**技术原理：**
- 不在源代码中植入后门
- 在源代码→可执行代码转换的最后一刻替换文件
- 修改已签名的软件代码而不破坏数字签名

**实现方式：**
```xml
<!-- 修改 BuildTask.xml 构建配置 -->
<Target Name="BeforeBuild">
  <!-- 正常构建步骤 -->
  <MSBuild Projects="Orion.Core.csproj" />
  
  <!-- 恶意步骤：替换编译后的文件 -->
  <ReplaceFile 
    Source="malicious/CiscoWebExService.dll" 
    Destination="build/CiscoWebExService.dll" />
</Target>
```

**恶意代码特征：**
- 仅 3,500 行，加密且精简
- 无人工痕迹（无西里尔字母变量名、无注释、无工具标记）
- 使用合法 SolarWinds 代码签名证书签名

| **奖金收入** | 超过 13 万美元 |
| **攻击手法** | 依赖混淆（Dependency Confusion） |

### 2.6.2 攻击原理

```
┌─────────────────────────────────────────────────────────────┐
│                  依赖混淆攻击原理                            │
├─────────────────────────────────────────────────────────────┤
│  1. 发现内部包名                                            │
│     - 通过错误消息、文档、泄露的 package.json                │
├─────────────────────────────────────────────────────────────┤
│  2. 在公共仓库注册同名包                                    │
│     - 使用更高版本号（如 99.99.99）                          │
│     - 包含恶意代码                                           │
├─────────────────────────────────────────────────────────────┤
│  3. 目标构建系统拉取依赖                                    │
│     - 包管理器未配置私有源优先                               │
│     - 从公共仓库拉取恶意包                                   │
├─────────────────────────────────────────────────────────────┤
│  4. 恶意代码执行                                            │
│     - postinstall 脚本执行                                   │
│     - 窃取 CI/CD 凭证                                        │
└─────────────────────────────────────────────────────────────┘
```

# 第三部分：附录

## 3.1 攻击技术分类

| 攻击技术 | 相关案例 |
|---------|---------|
| 构建系统入侵 | SolarWinds, CCleaner |
| 更新机制投毒 | NotPetya, SolarWinds |
| 依赖混淆 | Alex Birsan 研究 |
| 恶意 npm 包 | EventStream, Shai-Hulud |
| 社会工程 | EventStream |
| 条件触发后门 | Bybit |
| CI/CD 入侵 | Codecov |
| 自我传播蠕虫 | Shai-Hulud |

## 3.2 共同特征

1. **信任滥用**：利用用户对合法软件/更新的信任
2. **签名有效**：使用合法证书签名，绕过安全检测
3. **供应链传递**：通过供应链层层传递影响
4. **隐蔽性强**：潜伏期长，难以检测
5. **影响广泛**：单个攻击影响成千上万组织

## 3.3 防御建议

| 防御措施 | 描述 |
|---------|------|
| SBOM 管理 | 维护完整的软件物料清单 |
| 签名验证 | 验证所有制品的数字签名 |
| 依赖锁定 | 使用锁定文件确保依赖版本一致 |
| 私有源优先 | 配置包管理器优先使用私有源 |
| 构建隔离 | 使用隔离的构建环境 |
| 持续监控 | 监控依赖和构建过程 |
| 供应商评估 | 评估供应商的安全实践 |
