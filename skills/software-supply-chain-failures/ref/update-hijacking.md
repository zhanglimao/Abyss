# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

更新劫持是指攻击者通过篡改软件更新过程，将恶意代码作为合法更新分发给用户。

**本质问题**：
- 更新包未签名或签名验证缺失
- 更新通道未加密
- 更新服务器可被劫持

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-494 | 下载代码 without 完整性检查 |
| CWE-345 | 数据真实性验证不足 |
| CWE-829 | 从不可信控制域包含功能 |

## 2.2 攻击常见于哪些业务场景

| 场景 | 风险点 | 潜在危害 |
|-----|-------|---------|
| 桌面应用 | 自动更新 | 恶意软件分发 |
| 移动应用 | 热更新机制 | 代码执行 |
| IoT 设备 | 固件更新 | 设备沦陷 |
| 浏览器扩展 | 扩展更新 | 用户监控 |
| 企业软件 | 集中更新 | 大规模感染 |

## 2.4 漏洞利用方法

### 2.4.1 更新服务器劫持

```bash
# 1. 劫持更新服务器 DNS
# 2. 搭建恶意更新服务器
# 3. 返回恶意更新包

# 恶意更新包内容：
# - 正常功能 + 后门
# - 完全恶意代码
# - 间谍软件
```

### 2.4.2 更新包篡改

```bash
# 1. 拦截更新请求
# 2. 下载原始更新包
# 3. 解包并注入恶意代码
# 4. 重新打包（如无签名验证）
# 5. 发送给客户端
```

### 2.4.3 版本回滚攻击

```bash
# 1. 发现旧版本存在漏洞
# 2. 修改版本检查响应
# 3. 返回旧版本为"最新版本"
# 4. 用户被诱导降级
# 5. 利用已知漏洞攻击
```

### 2.4.4 供应链污染

```
攻击流程：
1. 攻陷更新服务器
2. 在所有更新中植入后门
3. 所有用户自动下载执行
4. 大规模感染

影响：
- 所有用户被影响
- 难以追溯源头
- 信任链被破坏
```

### 2.4.5 固件更新攻击

```bash
# IoT 设备固件更新通常防护较弱

# 1. 拦截固件更新
# 2. 修改固件植入后门
# 3. 设备刷入恶意固件
# 4. 设备被永久控制
```

## 2.5 漏洞利用绕过方法

### 2.5.1 签名验证绕过

```bash
# 如果签名验证存在缺陷：
# 1. 使用空签名
# 2. 使用无效签名
# 3. 移除签名检查代码
# 4. 利用签名算法弱点
```

### 2.5.2 证书验证绕过

```bash
# 如果证书验证不严格：
# 1. 使用自签名证书
# 2. 使用过期证书
# 3. 使用相似域名证书
# 4. 利用证书链缺陷
```

### 2.5.3 增量更新绕过

```bash
# 某些应用使用增量更新
# 只下载变更部分

# 如果增量更新验证较弱
# 可以只修改部分内容
```

## 3.3 修复建议

1. **代码签名** - 所有更新包必须签名
2. **HTTPS 传输** - 更新通道强制加密
3. **证书锁定** - 验证服务器证书
4. **版本检查** - 防止版本回滚
5. **分阶段发布** - 限制更新影响范围
