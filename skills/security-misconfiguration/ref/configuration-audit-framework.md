# 配置安全审计框架方法论

---

## 第一部分：文档概述

### 1.1 编写目的
为渗透测试人员提供一个系统化的配置安全审计框架，帮助组织和管理针对各类系统、应用、基础设施的配置审计工作。本框架提供标准化的流程、检查清单和最佳实践。

### 1.2 适用范围
- Web 应用服务器配置
- 数据库系统配置
- 网络设备配置
- 云基础设施配置
- 容器和编排平台
- 操作系统配置

### 1.3 读者对象
- 渗透测试工程师
- 安全审计人员
- 安全架构师
- 合规审计人员

## 第二部分：核心审计框架

### 专题：配置安全审计框架

#### 2.1 框架介绍

配置安全审计框架是一个系统化的方法，用于评估和验证 IT 系统的安全配置是否符合安全基线和最佳实践。

**框架核心组件：**

```
┌─────────────────────────────────────────────────────────┐
│                    配置安全审计框架                      │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  资产发现   │→│  基线对比   │→│  漏洞评估   │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         ↓                ↓                ↓             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  资产清单   │  │  配置基线   │  │  风险报告   │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         ↓                ↓                ↓             │
│  ┌─────────────────────────────────────────────┐       │
│  │              持续监控与改进                   │       │
│  └─────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘
```

**审计维度：**

| 维度 | 描述 | 检查内容 |
|-----|------|---------|
| **身份认证** | 用户认证机制 | 密码策略、MFA、会话管理 |
| **访问控制** | 权限管理机制 | RBAC、最小权限、审计日志 |
| **数据安全** | 数据保护机制 | 加密、脱敏、备份 |
| **网络安全** | 网络隔离与防护 | 防火墙、分段、WAF |
| **日志审计** | 日志记录与监控 | 日志完整性、告警、留存 |
| **补丁管理** | 漏洞修复机制 | 更新策略、补丁测试 |

#### 2.2 审计流程

**阶段一：准备阶段**

```
1. 确定审计范围
   - 系统/应用清单
   - 网络拓扑图
   - 数据流图

2. 收集参考资料
   - CIS Benchmarks
   - 厂商安全指南
   - 行业最佳实践

3. 准备工具
   - 扫描工具
   - 配置检查脚本
   - 报告模板
```

**阶段二：执行阶段**

```
1. 资产发现
   - 网络扫描
   - 服务识别
   - 应用指纹

2. 配置收集
   - 配置文件导出
   - 注册表/数据库查询
   - API 调用获取配置

3. 基线对比
   - 自动化扫描
   - 手工检查
   - 差异分析

4. 漏洞验证
   - 漏洞利用测试
   - 配置绕过测试
   - 影响评估
```

**阶段三：报告阶段**

```
1. 风险评级
   - CVSS 评分
   - 业务影响分析
   - 修复优先级

2. 报告编写
   - 执行摘要
   - 详细发现
   - 修复建议

3. 结果汇报
   - 技术团队汇报
   - 管理层汇报
   - 整改计划
```

#### 2.3 通用配置检查清单

**身份认证配置：**

| 检查项 | 安全要求 | 检查方法 |
|-------|---------|---------|
| 密码复杂度 | 最小 12 位，包含大小写、数字、特殊字符 | 查看密码策略配置 |
| 密码历史 | 禁止复用最近 5 次密码 | 查看密码历史策略 |
| 账户锁定 | 5 次失败后锁定 15 分钟 | 查看锁定策略 |
| MFA 启用 | 管理员和远程访问强制 MFA | 查看 MFA 配置 |
| 默认凭证 | 无默认或弱凭证 | 暴力破解测试 |

**访问控制配置：**

| 检查项 | 安全要求 | 检查方法 |
|-------|---------|---------|
| 最小权限 | 用户仅有必要权限 | 权限审计 |
| 职责分离 | 关键操作需多人审批 | 流程审查 |
| 会话管理 | 空闲超时≤15 分钟 | 查看会话配置 |
| 管理访问 | 管理界面限制 IP 访问 | 查看访问控制列表 |

**网络安全配置：**

| 检查项 | 安全要求 | 检查方法 |
|-------|---------|---------|
| 端口开放 | 仅开放必要端口 | 端口扫描 |
| 服务暴露 | 内部服务不对外暴露 | 网络扫描 |
| 加密传输 | 使用 TLS 1.2+ | SSL 扫描 |
| 网络分段 | 不同安全域隔离 | 网络拓扑审查 |

**日志审计配置：**

| 检查项 | 安全要求 | 检查方法 |
|-------|---------|---------|
| 日志开启 | 所有安全事件记录 | 查看日志配置 |
| 日志内容 | 包含时间、用户、操作、结果 | 日志抽样检查 |
| 日志保护 | 防篡改、集中存储 | 日志完整性检查 |
| 日志留存 | 至少 6 个月 | 查看留存策略 |

#### 2.4 平台特定检查清单

**Linux 系统配置：**

```bash
# 1. 检查密码策略
cat /etc/login.defs | grep -E "PASS_MAX_DAYS|PASS_MIN_LEN"
cat /etc/pam.d/common-password

# 2. 检查 SSH 配置
cat /etc/ssh/sshd_config | grep -E "PermitRootLogin|PasswordAuthentication"

# 3. 检查防火墙配置
iptables -L -n
firewall-cmd --list-all

# 4. 检查系统更新
cat /etc/os-release
uname -r
```

**Windows 系统配置：**

```powershell
# 1. 检查密码策略
net accounts

# 2. 检查本地用户
net user

# 3. 检查服务配置
Get-Service | Where-Object {$_.Status -eq "Running"}

# 4. 检查防火墙配置
Get-NetFirewallRule | Where-Object {$_.Enabled -eq True}
```

**Docker 配置：**

```bash
# 1. 检查 Docker 配置
docker info
docker config ls

# 2. 检查容器配置
docker ps --format "table {{.Names}}\t{{.Image}}\t{{.Status}}"
docker inspect <container>

# 3. 运行 Docker Bench
docker run --rm \
  --net host --pid host --userns host \
  --cap-add audit_control \
  -v /etc:/etc:ro \
  -v /var/lib/docker:/var/lib/docker:ro \
  docker/docker-bench-security
```

**Kubernetes 配置：**

```bash
# 1. 检查集群配置
kubectl cluster-info
kubectl get nodes

# 2. 检查 RBAC 配置
kubectl get clusterroles
kubectl get clusterrolebindings

# 3. 检查 Pod 安全
kubectl get pods --all-namespaces -o yaml

# 4. 运行 Kube-bench
kube-bench
```
