# 访问控制测试框架方法论

---

# 第一部分：文档概述

## 1.1 编写目的

本文档旨在为渗透测试人员提供一套系统化的访问控制测试框架，确保全面覆盖各类访问控制漏洞。

## 1.2 适用范围

本文档适用于所有需要测试访问控制的 Web 应用、API 接口和移动应用。

## 1.3 读者对象

本文件主要面向执行渗透测试任务的安全工程师、安全分析师。

---

# 第二部分：核心渗透技术专题

## 2.1 测试框架概述

### 访问控制类型

| 类型 | 描述 | 测试重点 |
|-----|------|---------|
| 垂直访问控制 | 不同角色间的访问限制 | 权限提升 |
| 水平访问控制 | 同角色不同用户间的限制 | IDOR |
| 上下文访问控制 | 基于上下文的访问限制 | 状态/流程绕过 |

### 测试阶段

```
┌─────────────────────────────────────────────────────────┐
│                   访问控制测试流程                       │
├─────────────────────────────────────────────────────────┤
│  1. 信息收集 → 2. 威胁建模 → 3. 漏洞检测 → 4. 漏洞利用  │
│       ↓              ↓              ↓              ↓     │
│   识别资源      绘制权限矩阵    系统化测试      验证影响  │
└─────────────────────────────────────────────────────────┘
```

## 2.2 信息收集阶段

### 2.2.1 资源识别

```
识别以下资源类型：
- 用户数据（个人资料、订单、消息）
- 管理功能（用户管理、配置管理）
- API 端点（REST、GraphQL）
- 文件资源（文档、图片）
- 系统功能（日志、备份）
```

### 2.2.2 角色识别

```
识别系统中的角色：
- 匿名用户
- 普通用户
- VIP 用户
- 管理员
- 超级管理员
```

### 2.2.3 权限矩阵绘制

| 资源/功能 | 匿名 | 用户 | VIP | 管理员 |
|---------|-----|------|-----|-------|
| 查看个人资料 | × | ✓ | ✓ | ✓ |
| 修改个人资料 | × | ✓ | ✓ | ✓ |
| 查看他人资料 | × | × | × | ✓ |
| 用户管理 | × | × | × | ✓ |
| 系统配置 | × | × | × | ✓ |

## 2.3 威胁建模阶段

### 2.3.1 攻击场景识别

```
针对每个资源/角色组合，识别潜在攻击场景：
1. 未认证访问
2. 水平越权访问
3. 垂直越权访问
4. 上下文绕过
```

### 2.3.2 测试用例设计

```
为每个场景设计测试用例：
- 输入：请求详情
- 预期：正确的访问控制响应
- 实际：测试实际响应
```

## 2.4 漏洞检测阶段

### 2.4.1 自动化测试

```bash
# 使用 AuthMatrix 等工具
# 批量测试权限矩阵

# 使用 Burp Suite
# 配置权限测试插件
```

### 2.4.2 手工测试

```
针对关键功能进行手工测试：
1. 修改请求参数
2. 更换认证凭据
3. 检查响应差异
```

## 2.5 漏洞利用阶段

### 2.5.1 影响验证

```
验证发现漏洞的实际影响：
- 可访问的数据范围
- 可执行的操作
- 对业务的影响
```

### 2.5.2 证据收集

```
收集漏洞利用证据：
- 请求/响应截图
- 数据泄露证明
- 操作执行证明
```

# 第三部分：附录

## 3.1 访问控制测试检查清单

```
□ 识别所有资源和功能
□ 识别所有用户角色
□ 绘制完整权限矩阵
□ 测试未认证访问
□ 测试水平越权
□ 测试垂直越权
□ 测试上下文绕过
□ 测试 API 访问控制
□ 测试文件访问控制
□ 记录所有测试结果
```

## 3.2 常用工具

| 工具 | 用途 | 链接 |
|-----|------|------|
| AuthMatrix | 权限矩阵测试 | https://github.com/SecurityInnovation/AuthMatrix |
| Burp Suite | 请求篡改测试 | https://portswigger.net/burp |
| OWASP ZAP | 自动化扫描 | https://www.zaproxy.org/ |

## 3.3 修复建议

1. **统一访问控制** - 使用统一的授权机制
2. **默认拒绝** - 默认拒绝所有访问
3. **服务端验证** - 所有检查在服务端执行
4. **审计日志** - 记录所有访问尝试
