# 访问控制测试框架方法论

---

# 第一部分：文档概述

## 1.1 编写目的

本文档旨在为渗透测试人员提供一套系统化的访问控制测试框架，确保全面覆盖各类访问控制漏洞，基于 OWASP WSTG 和 OWASP Top 10 2025 标准。

## 1.2 适用范围

本文档适用于所有需要测试访问控制的 Web 应用、API 接口和移动应用，特别是：
- 多角色权限系统
- RESTful/GraphQL API 服务
- 多租户 SaaS 应用
- 微服务架构系统

## 1.3 读者对象

本文件主要面向执行渗透测试任务的安全工程师、安全分析师。

## 2.6 OWASP WSTG 授权测试方法

### 2.6.1 目录遍历文件包含测试 (WSTG-ATHZ-01)

**测试目标：** 检测攻击者能否通过操纵文件路径访问未授权文件

**测试步骤：**
1. 识别接受文件路径参数的功能点
2. 尝试使用 `../` 序列遍历目录
3. 测试 URL 编码的遍历序列 (`%2e%2e%2f`)
4. 尝试访问系统敏感文件 (`/etc/passwd`, `C:\Windows\System32\config\SAM`)

**常用 Payload：**
```
../../../etc/passwd
....//....//....//etc/passwd
%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
..%c0%af..%c0%af..%c0%afetc/passwd
```

**工具：** Burp Suite, OWASP ZAP, DirBuster

### 2.6.2 绕过授权架构测试 (WSTG-ATHZ-02)

**测试目标：** 检测能否绕过访问控制机制

**测试方法：**
1. **URL 直接访问测试** - 直接输入受保护资源的 URL
2. **HTTP 方法绕过** - 将 POST 改为 GET/PUT/DELETE
3. **参数篡改** - 修改用户 ID、角色参数
4. **强制浏览** - 使用爬虫枚举管理页面

**测试步骤：**
1. 以低权限用户登录
2. 记录所有请求和响应
3. 尝试访问高权限资源
4. 修改请求中的身份标识参数
5. 测试未公开的管理接口

### 2.6.3 权限提升测试 (WSTG-ATHZ-03)

**测试目标：** 检测用户能否获取超出其授权范围的权限

**类型：**
- **垂直权限提升** - 低权限用户获取高权限
- **水平权限提升** - 访问同级其他用户的资源

**测试步骤：**
1. 创建多个不同权限级别的测试账户
2. 捕获低权限用户的请求
3. 尝试修改以下参数：
   - 用户 ID (`user_id=1001` → `user_id=1`)
   - 角色标识 (`role=user` → `role=admin`)
   - 组标识 (`group_id=100` → `group_id=1`)
4. 重放修改后的请求
5. 验证是否获得更高权限

**常见攻击向量：**
```
POST /api/updateProfile
{"userId": "123", "isAdmin": false}
→ 修改为 {"userId": "123", "isAdmin": true}
```

### 2.6.4 不安全直接对象引用测试 (WSTG-ATHZ-04)

**测试目标：** 检测是否可通过修改对象引用访问未授权数据

**测试步骤：**
1. 识别引用内部对象的参数（ID、文件名、键值）
2. 以用户 A 身份访问资源，记录对象 ID
3. 以用户 B 身份重放请求，修改对象 ID 为用户 A 的资源
4. 验证是否可访问其他用户的数据

**测试案例：**
```
# 原始请求（用户 123 访问自己的订单）
GET /api/orders/12345

# 攻击请求（尝试访问用户 123 的订单）
GET /api/orders/12346
```

**自动化测试工具：** Burp Suite Intruder, AuthMatrix, Autorize

### 2.6.5 OAuth 弱点测试 (WSTG-ATHZ-05)

#### OAuth 授权服务器弱点测试

**测试要点：**
1. **重定向 URI 验证** - 测试开放重定向
2. **授权码劫持** - 拦截授权码
3. **状态参数验证** - 检查 CSRF 保护
4. **令牌安全性** - 验证访问令牌的强度和有效期

**测试步骤：**
```
1. 发起 OAuth 授权请求
2. 修改 redirect_uri 参数为攻击者控制的域名
3. 检查是否接受任意重定向 URI
4. 尝试重放授权码
5. 测试令牌的刷新机制
```

#### OAuth 客户端弱点测试

**测试要点：**
1. **客户端凭证存储** - 检查硬编码密钥
2. **隐式授权流** - 验证令牌泄露风险
3. **范围验证** - 测试越权访问范围
4. **令牌存储安全** - 检查本地存储安全性

### 2.6.6 敏感信息暴露测试

**测试目标：** 检测应用是否暴露敏感信息给未授权用户

**测试场景：**
1. **错误信息泄露** - 详细错误消息暴露系统信息
2. **响应头泄露** - Server、X-Powered-By 等头信息
3. **注释泄露** - HTML/JS 注释中的敏感信息
4. **元数据泄露** - 文件元数据包含敏感信息

**测试步骤：**
```
1. 触发各种错误条件
2. 检查响应头信息
3. 查看页面源代码注释
4. 检查文件元数据
```

### 2.6.7 会话管理测试

**测试目标：** 检测会话管理机制的安全性

**测试要点：**
1. **会话令牌生成** - 检查随机性和可预测性
2. **会话令牌传输** - 检查是否使用 HTTPS
3. **会话超时** - 检查超时机制
4. **会话终止** - 检查登出后会话是否失效

**测试步骤：**
```
1. 分析会话令牌格式
2. 测试会话固定攻击
3. 测试会话超时
4. 测试登出功能
```

## 3.3 修复建议

1. **统一访问控制** - 使用统一的授权机制
2. **默认拒绝** - 默认拒绝所有访问
3. **服务端验证** - 所有检查在服务端执行
4. **审计日志** - 记录所有访问尝试
5. **最小权限原则** - 只授予必要的权限
6. **定期审计** - 定期审查访问控制策略

## 3.4 OWASP Top 10:2025 A01 测试要点

### 3.4.1 常见漏洞类型测试

| 漏洞类型 | 测试方法 | CWE 映射 |
|---------|---------|---------|
| 违反最小权限原则 | 测试默认访问权限 | CWE-284 |
| URL/状态/HTML 绕过 | 修改请求参数测试 | CWE-639 |
| 不安全直接对象引用 | 遍历对象 ID 测试 | CWE-639 |
| API 缺少访问控制 | 测试 POST/PUT/DELETE | CWE-285 |
| 权限提升 | 测试未登录/越权操作 | CWE-285 |
| 元数据操纵 | 修改 JWT/Cookie/隐藏字段 | CWE-287 |
| CORS 配置错误 | 测试 Origin 头反射 | CWE-942 |
| 强制浏览 | 枚举隐藏 URL | CWE-425 |

### 3.4.2 防御措施验证

```
验证以下防御措施是否到位：
□ 默认拒绝所有访问（除公共资源）
□ 访问控制机制统一且重用
□ 记录所有权验证（非任意 CRUD）
□ 业务限制由域模型执行
□ 目录列表已禁用
□ 访问控制失败有日志记录
□ API 和控制器有速率限制
□ 会话标识符注销后失效
□ JWT 设置为短有效期
```
