## 第二部分：核心渗透技术专题

### 2.1 技术介绍

UEBA（User and Entity Behavior Analytics）通过分析用户和实体的行为模式来检测异常活动。UEBA 规避技术是指攻击者通过模仿正常行为、分散攻击活动等手段绕过行为分析检测。

**核心原理：**
- **基线模仿**：攻击行为模仿正常用户行为基线
- **慢速攻击**：将攻击分散到长时间窗口避免触发异常检测
- **合法凭证滥用**：使用窃取的合法凭证进行攻击
- **业务逻辑滥用**：利用正常业务流程执行恶意操作

### 2.2 攻击常见于哪些业务场景

| **业务场景** | **功能示例** | **UEBA 检测点** | **规避机会** |
| :--- | :--- | :--- | :--- |
| **数据访问** | 数据库查询 | 访问模式异常 | 模仿正常查询模式 |
| **文件下载** | 文档下载 | 下载量异常 | 分散到多天 |
| **登录行为** | 系统登录 | 时间/地点异常 | 使用被盗凭证 |
| **权限使用** | 特权操作 | 权限滥用 | 在职责范围内 |
| **API 调用** | API 访问 | 调用频率异常 | 低频慢速调用 |

### 2.3 漏洞探测方法

#### 2.3.1 黑盒测试

**行为基线探测：**
```bash
# 观察正常用户行为模式
# 1. 工作时间
# 记录正常用户的登录时间分布

# 2. 访问资源
# 记录正常用户访问的资源类型

# 3. 操作频率
# 记录正常用户的操作频率

# 4. 数据量
# 记录正常用户的数据访问量
```

**UEBA 检测阈值探测：**
```bash
# 探测数据下载阈值
# 逐步增加下载量观察是否触发告警

for size in 10 50 100 500 1000; do
    curl "http://target/api/export?limit=$size"
    # 检查是否触发告警
done

# 探测访问频率阈值
for i in {1..100}; do
    curl "http://target/api/data"
    sleep 1
    # 检查是否触发告警
done
```

#### 2.3.2 白盒测试

**UEBA 配置审计：**
```yaml
# UEBA 规则配置示例
# 检查检测阈值和基线

user_behavior_baseline:
  login_hours:
    normal_range: "08:00-20:00"
    anomaly_threshold: 22:00-06:00
  data_access:
    normal_daily_mb: 100
    anomaly_threshold_mb: 500
  api_calls:
    normal_hourly: 50
    anomaly_threshold: 200
```

### 2.4 漏洞利用方法

#### 2.4.1 行为基线模仿

```bash
# 模仿正常用户行为模式

# 1. 工作时间攻击
# 只在正常工作时间（9:00-18:00）进行攻击

# 2. 正常操作频率
# 模仿正常用户的操作间隔
while true; do
    curl "http://target/api/data"
    # 随机间隔 30-120 秒，模仿人工操作
    sleep $((30 + RANDOM % 90))
done

# 3. 正常数据访问量
# 每次访问限制在正常范围内
curl "http://target/api/export?limit=50"  # 正常范围
```

#### 2.4.2 合法凭证滥用

```bash
# 使用窃取的合法凭证
# UEBA 难以检测，因为凭证是合法的

curl -H "Authorization: Bearer $STOLEN_TOKEN" \
     "http://target/api/sensitive-data"

# 在凭证所有者的正常行为范围内操作
# 例如：如果用户通常访问财务报表
# 攻击者也主要访问财务报表，夹杂敏感操作
```

#### 2.4.3 业务逻辑滥用

```bash
# 利用正常业务流程执行恶意操作

# 例如：通过正常的数据导出功能窃取数据
# 1. 使用合法的报表功能
curl "http://target/reports/sales?range=last_year"

# 2. 使用合法的数据同步功能
curl "http://target/api/sync?target=backup_server"

# 3. 使用合法的 API 集成功能
curl "http://target/api/integration/export?format=json"
```

### 2.5 漏洞利用绕过方法

#### 2.5.1 绕过时间分析

```bash
# UEBA 通常分析时间模式
# 绕过方法：

# 1. 时间分散
# 将攻击分散到数天或数周
# 每天只执行少量操作

# 2. 时间模仿
# 在目标用户通常活跃的时间操作
# 例如：如果用户通常在上午 10 点登录
# 攻击也在上午 10 点进行
```

#### 2.5.2 绕过地点分析

```bash
# UEBA 通常分析地理位置
# 绕过方法：

# 1. 使用目标用户常用的地理位置
# 通过该地区的代理或 VPN

# 2. 模仿出差模式
# 如果用户有出差历史
# 从出差地点发起攻击
```

#### 2.5.3 绕过设备指纹

```bash
# UEBA 通常分析设备特征
# 绕过方法：

# 1. 使用被盗设备的指纹
# User-Agent、屏幕分辨率、时区等

curl -H "User-Agent: $VICTIM_UA" \
     -H "Accept-Language: $VICTIM_LANG" \
     "http://target/api/data"

# 2. 使用常见设备指纹
# 使用最常见的浏览器配置
```
