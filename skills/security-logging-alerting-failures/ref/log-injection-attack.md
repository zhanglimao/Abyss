# 第二部分：核心渗透技术专题

## 2.1 技术介绍

### 漏洞原理

日志注入是指攻击者向日志系统注入伪造或恶意的日志条目，用于混淆审计、隐藏攻击痕迹或触发日志处理系统的漏洞。

**本质问题**：
- 用户输入未编码直接写入日志
- 日志格式可被操纵
- 日志处理系统存在漏洞

### 常见 CWE 映射

| CWE 编号 | 描述 |
|---------|------|
| CWE-117 | 日志输出中和不当 |
| CWE-532 | 敏感信息写入日志 |

## 2.2 攻击常见于哪些业务场景

| 业务场景 | 风险点 | 潜在危害 |
|---------|-------|---------|
| 登录功能 | 用户名记录 | 注入伪造登录记录 |
| 搜索功能 | 搜索词记录 | 注入恶意内容 |
| 评论系统 | 评论内容记录 | 存储 XSS 到日志 |
| API 接口 | 请求参数记录 | 注入伪造审计记录 |
| 错误处理 | 错误信息记录 | 泄露敏感信息 |

## 2.3 漏洞发现方法

### 2.3.1 日志注入点识别

```bash
# 识别记录用户输入的功能
# - 登录用户名
# - 搜索关键词
# - 表单输入
# - URL 参数
# - HTTP 头（User-Agent, Referer）
```

### 2.3.2 换行符注入测试

```bash
# 测试 CRLF 注入
# 在输入中包含换行符

# HTTP 头注入
User-Agent: Mozilla/5.0
2025-01-15 10:00:00 SUCCESS Login as admin from 127.0.0.1

# 如果日志系统不编码，会创建伪造日志条目
```

### 2.3.3 日志内容分析

```bash
# 如果有权限访问日志文件
# 检查是否包含未编码的用户输入
# 检查是否可以注入伪造条目

tail -f /var/log/application.log
```

## 2.4 漏洞利用方法

### 2.4.1 伪造审计记录

```
攻击者注入以下内容到日志：

2025-01-15 10:00:00 INFO User login: admin
2025-01-15 10:00:01 INFO Access granted to /admin
[注入] 2025-01-15 10:00:02 SUCCESS Login as admin from 127.0.0.1
2025-01-15 10:00:03 INFO User logout

目的：
- 混淆真实攻击时间
- 伪造成功登录记录
- 误导安全分析
```

### 2.4.2 日志 XSS 攻击

```javascript
// 如果日志查看器是 Web 界面
// 注入 XSS Payload 到日志

Username: <script>alert(document.cookie)</script>

// 当管理员查看日志时，XSS 执行
```

### 2.4.3 日志洪水攻击

```bash
# 大量请求产生大量日志
# 填满磁盘空间
# 导致服务不可用

for i in {1..10000}; do
    curl https://target.com/search?q=test$i &
done
```

### 2.4.4 敏感信息泄露

```bash
# 触发错误使敏感信息记录到日志
# 然后尝试访问日志文件

# 常见泄露：
# - SQL 查询（含凭证）
# - 完整请求 URL（含 Token）
# - 堆栈跟踪（含路径）
```

## 2.5 漏洞利用绕过方法

### 2.5.1 日志格式绕过

```bash
# 测试不同日志格式
# JSON 日志
# Syslog 格式
# Common Log Format

# 某些格式可能更容易注入
```

### 2.5.2 编码绕过

```bash
# 如果某些字符被编码
# 尝试 Unicode 编码
# 尝试 HTML 实体编码

Username: \u000a2025-01-15...
```

### 2.5.3 时间戳混淆

```
注入伪造的时间戳：
- 过去的日期（混淆调查时间线）
- 未来的日期（使日志排序混乱）
```

# 第三部分：附录

## 3.1 日志注入测试检查清单

```
□ 测试换行符注入
□ 测试 HTML/XSS 注入
□ 测试时间戳注入
□ 测试日志洪水
□ 检查敏感信息记录
□ 检查日志查看器安全
□ 测试日志完整性保护
```

## 3.2 常用 Payload

| 类型 | Payload | 说明 |
|-----|---------|------|
| 换行符 | `%0d%0a` | CRLF 注入 |
| XSS | `<script>alert(1)</script>` | 日志 XSS |
| 时间戳 | `2025-01-15 10:00:00` | 伪造时间 |
| 状态 | `SUCCESS` | 伪造成功状态 |

## 3.3 修复建议

1. **输入编码** - 写入日志前编码所有用户输入
2. **结构化日志** - 使用 JSON 等结构化格式
3. **日志完整性** - 实施日志签名/哈希
4. **访问控制** - 限制日志文件访问权限
